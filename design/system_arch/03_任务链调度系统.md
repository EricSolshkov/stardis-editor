# 任务链调度系统

> **归档层次**：系统架构，面向全局理解与跨系统集成。
> 详细实现参见 [`design/task_runner/`](../task_runner/)。

---

## 1. 系统定位

任务链调度系统将场景配置从"描述"转化为"执行"：用户在编辑器中配置好几何体和边界条件后，通过任务队列驱动 Stardis/HTPP 可执行文件完成仿真计算。

系统取代了 v1 独立控制面板的手动工作流，深度集成于 Scene Editor v2：
- 任务通过**名称引用**场景中的 Camera / Probe，不复制数据
- 任务配置存储于 `.stardis_project.json`，随工程一同持久化
- 支持 HTPP 自动依赖 Stardis IR 输出的 `.ht` 文件

---

## 2. 系统架构

```
SceneEditor（主窗口）
    │
    │ 用户点击"运行"
    ▼
TaskRunner (QObject)              ← 核心调度器
    ├── validate_queue()          ← 运行前引用校验
    ├── resolve_all()             ← 名称引用 → ResolvedTask（参数快照）
    ├── run_queue()               ← 顺序执行
    │     ├── _run_next()
    │     │     ├── CommandBuilder.build_stardis/htpp()   ← 参数 → CLI 列表
    │     │     ├── QProcess.start()                      ← 启动子进程
    │     │     └── _on_finished()                        ← 完成回调
    │     └── _handle_error()     ← 错误策略分发（重试/跳过/暂停/取消）
    └── run_single(task_id)       ← 单任务独立执行
```

## 3. 任务类型与计算模式

### 3.1 任务类型（TaskType）

| 类型 | 可执行文件 | 说明 |
|------|-----------|------|
| `STARDIS` | stardis | 仿真计算（探针/场/IR 渲染） |
| `HTPP` | htpp | IR 图像后处理（`.ht` → 图像/伪色图） |

### 3.2 Stardis 计算模式（ComputeMode）

| 模式 | CLI 参数 | 输入引用 | 产出 |
|------|---------|---------|------|
| `PROBE_SOLVE` | `-p/-P/-f` | `Probe` 列表 | stdout 数值（T / SE / N） |
| `FIELD_SOLVE` | `-m/-s/-S/-F` | Body 名 / STL 文件 | stdout 数值 / VTK POLYDATA |
| `IR_RENDER` | `-R spp=:img=:fov=:pos=:tgt=:up=` | `IRCamera` | `.ht` 文件 |

### 3.3 HTPP 处理模式（HtppMode）

| 模式 | CLI 参数 | 说明 |
|------|---------|------|
| `IMAGE` | `-i exposure[:white=...]` | 生成可视化图像 |
| `MAP` | `-m pixcpnt=:palette=:range=:gnuplot` | 生成伪色温度图 |

### 3.4 HTPP 输入来源（InputSource）

| 来源类型 | 说明 |
|---------|------|
| `InputFromTask(task_id)` | 自动引用队列中某 IR_RENDER 任务的 `.ht` 输出 |
| `InputFromFile(file_path)` | 直接指定已有 `.ht` 文件路径 |

## 4. 引用语义与求值时机

```
TaskQueue（任务配置，含名称引用）
    │
    │  queue.start() 时触发 resolve_all()
    ▼
ResolvedTask（参数快照，已将名称替换为具体数值）
    │
    │  运行期间不再跟踪场景变化
    ▼
CommandBuilder → CLI 参数列表 → QProcess.start()
```

**名称引用示例**：
- `stardis_params.camera_ref = "Camera1"` → 求值时读取 `SceneModel.cameras["Camera1"]` 的位置/方向/分辨率
- `htpp_params.input_source = InputFromTask("a1b2c3")` → 求值时查找该任务 `id` 对应的 `.ht` 输出路径

## 5. 错误处理策略（ErrorPolicy）

| 策略 | `after_retries_exhausted` 值 | 行为 |
|------|----------------------------|------|
| 取消队列 | `CANCEL` | 停止整个队列，标记剩余任务为已跳过 |
| 跳过继续 | `SKIP` | 跳过失败任务，继续执行后续任务 |
| 暂停等待 | `PAUSE` | 暂停队列，等待用户手动决定（重试/跳过/取消） |

`retry_count`：在触发 `after_retries_exhausted` 前的最大重试次数，默认 0（不重试）。

## 6. CLI 参数构建规则（CommandBuilder）

### 6.1 非默认值原则

**仅当参数值非默认时才生成对应标志**，与 C 源码解析逻辑一致，避免产生冗余参数。

### 6.2 Stardis 参数顺序

```
stardis -M model_file [-t threads] [-V verbosity]
  [-R spp=S:img=WxH:fov=F:pos=x,y,z:tgt=x,y,z:up=x,y,z]   # IR_RENDER
  [-p x,y,z[,t]] [-P x,y,z:side] [-f x,y,z] [-n samples]    # PROBE_SOLVE
  [-m name] / [-s stl] / [-S stl] / [-F stl] [-n samples]    # FIELD_SOLVE
  [-a algo] [-o order] [-I t0] [-i] [-e] [-x in] [-X out]    # 高级参数
```

### 6.3 HTPP 参数顺序

```
htpp [-f] [-v] [-i image_opts] [-m map_opts] [-o output] [-t threads] input.ht
```

IMAGE 模式 `-i` 参数组合规则：
- `exposure!=1.0` → `exposure=X`
- `white_scale is not None` → `white=Y`
- 全默认 → 不传 `-i` 标志

MAP 模式 `-m` 参数组合规则：
- `pixcpnt!=0` → `pixcpnt=X`
- `palette!="inferno"` → `palette=Y`
- `range is not None` → `range=A,B`
- `gnuplot=True` → `gnuplot`
- 全默认 → 不传 `-m` 标志

## 7. UI 集成：场景树与属性面板

### 7.1 场景树中的任务节点

```
SceneTree
└── Tasks/                          ← 顶层分组节点（与 Bodies / Probes 同级）
    ├── [1] IR渲染-Camera1           (Stardis/IR)
    ├── [2] 后处理-Camera1           (HTPP/Image ← [1])
    └── [3] 探针计算-P1,P2           (Stardis/Probe)
```

**任务节点着色**：

| 状态 | 颜色 |
|------|------|
| 正常 | 默认 |
| 禁用 | 灰色 |
| 引用缺失 | 橙色（warn） |
| 运行中 | 蓝色 |
| 完成 | 绿色 |
| 失败 | 红色 |

### 7.2 属性面板扩展

| 页面 | 触发 | 内容 |
|------|------|------|
| `TaskQueueEditor` | 选中 Tasks/ 分组 | 错误策略、队列级环境变量、任务列表总览、运行全部 |
| `TaskEditor` | 选中具体任务节点 | 名称/类型/启用、执行环境（exe/工作目录）、Stardis 或 HTPP 参数区、命令预览 |

### 7.3 右键快捷创建

| 右键对象 | 菜单项 | 创建结果 |
|---------|--------|---------|
| Camera 节点 | 创建渲染任务组 | 1× Stardis IR_RENDER + 1× HTPP IMAGE（自动依赖前者） |
| Probe 节点 | 创建探针计算任务 | 1× Stardis PROBE_SOLVE（引用该探针） |

## 8. 持久化格式（`.stardis_project.json` 扩展）

任务配置存储于 `task_queue` 字段，附加在现有工程元数据中：

```json
{
  "probes": [...],
  "cameras": [...],
  "task_queue": {
    "error_policy": { "retry_count": 0, "after_retries_exhausted": "cancel" },
    "env_vars": { "OMP_NUM_THREADS": "4" },
    "tasks": [
      {
        "id": "<uuid4>",
        "name": "IR渲染-Camera1",
        "task_type": "stardis",
        "enabled": true,
        "exe_ref": "stardis-v3",
        "working_dir": "",
        "compute_mode": "ir_render",
        "stardis_params": {
          "model_file": "",
          "samples": 1000000,
          "threads": 4,
          "camera_ref": "Camera1",
          ...
        }
      },
      {
        "id": "<uuid4>",
        "name": "后处理-Camera1",
        "task_type": "htpp",
        "htpp_mode": "image",
        "input_source": { "type": "from_task", "task_id": "<上一任务uuid>" },
        "htpp_params": { "exposure": 1.0, "threads": 4, ... }
      }
    ]
  }
}
```
