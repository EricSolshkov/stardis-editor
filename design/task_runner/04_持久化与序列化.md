## 1. 存储位置

任务配置**绑定项目**，存储在与场景文件同目录的 `.stardis_project.json` 中。

```
project_dir/
├── scene.txt
├── S_Wall.stl
├── B_zone1.stl
└── .stardis_project.json      ← tasks 存储在此
```

## 3. 序列化规则

### 3.1 字段约定

| 规则 | 说明 |
|------|------|
| `null` vs 省略 | 不适用的字段写 `null`（如 stardis 任务的 `htpp_params`），不省略 key |
| 空字符串 | `""` 表示"未填写"（如 `working_dir`），运行时使用默认值（场景目录） |
| 枚举 | 存储为小写字符串值（`"ir_render"`, `"image"`, `"cancel"` 等） |
| exe_ref | 优先存储标签名（如 `"stardis-v3"`）；若无标签则存储绝对路径作为回退 |
| 过程/结果文件路径 | `output_redirect`、`InputFromFile.file_path`、`HtppParams.output_file` 等引用 `.ht`/`.ppm` 的路径字段，一律使用**相对于 `working_dir` 的相对路径**（通常为纯文件名） |
| `working_dir` | 绝对路径。为空时运行时默认为场景目录 |
| UUID | `uuid4` 十六进制字符串（含连字符） |

### 3.2 InputSource 多态序列化

```json
// InputFromTask
{"type": "from_task", "task_id": "a1b2c3d4-..."}

// InputFromFile
{"type": "from_file", "file_path": "result.ht"}
```

### 3.3 向后兼容

- 加载不含 `task_queue` 字段的旧项目文件时，初始化为空 `TaskQueue()`
- 旧版编辑器忽略 `task_queue` 字段（JSON 的容错特性）

## 5. 序列化辅助函数

```python
def _task_queue_to_dict(queue: TaskQueue) -> dict:
    return {
        "error_policy": {
            "retry_count": queue.error_policy.retry_count,
            "after_retries_exhausted": queue.error_policy.after_retries_exhausted.value,
        },
        "env_vars": queue.env_vars,
        "tasks": [_task_to_dict(t) for t in queue.tasks],
    }

def _task_to_dict(task: Task) -> dict:
    d = {
        "id": task.id,
        "name": task.name,
        "task_type": task.task_type.value,
        "enabled": task.enabled,
        "exe_ref": task.exe_ref,
        "working_dir": task.working_dir,
        "env_vars": task.env_vars,
        "compute_mode": task.compute_mode.value if task.compute_mode else None,
        "stardis_params": _stardis_params_to_dict(task.stardis_params) if task.stardis_params else None,
        "htpp_mode": task.htpp_mode.value if task.htpp_mode else None,
        "htpp_params": _htpp_params_to_dict(task.htpp_params) if task.htpp_params else None,
        "input_source": _input_source_to_dict(task.input_source) if task.input_source else None,
        "output_redirect": task.output_redirect,
    }
    return d

def _input_source_to_dict(src: InputSource) -> dict:
    if isinstance(src, InputFromTask):
        return {"type": "from_task", "task_id": src.task_id}
    elif isinstance(src, InputFromFile):
        return {"type": "from_file", "file_path": src.file_path}

def _dict_to_input_source(d: dict) -> InputSource:
    if d["type"] == "from_task":
        return InputFromTask(task_id=d["task_id"])
    elif d["type"] == "from_file":
        return InputFromFile(file_path=d["file_path"])
```

反序列化函数 `_dict_to_task`、`_dict_to_task_queue` 等对称实现，此处省略。

## 7. 引用完整性维护

### 7.1 Camera/Probe 删除时

当用户删除 Camera 或 Probe 时，`SceneEditor` 检查 `task_queue.tasks` 中是否有引用：

```python
def _check_task_refs_before_delete(self, obj_type: str, name: str) -> bool:
    """检查并提示任务引用，返回是否继续删除。"""
    affected = []
    for task in self._model.task_queue.tasks:
        if obj_type == "camera" and task.stardis_params and task.stardis_params.camera_ref == name:
            affected.append(task)
        elif obj_type == "probe" and task.stardis_params and name in task.stardis_params.probe_refs:
            affected.append(task)

    if affected:
        names = ", ".join(t.name for t in affected)
        reply = QMessageBox.warning(self, "引用冲突",
            f"以下任务引用了 {name}: {names}\n删除后这些任务将标记为警告状态。\n继续？",
            QMessageBox.Yes | QMessageBox.No)
        return reply == QMessageBox.Yes
    return True
```

### 7.2 Camera/Probe 重命名时

```python
def _update_task_refs(self, obj_type: str, old_name: str, new_name: str):
    """级联更新任务中的引用名称。"""
    for task in self._model.task_queue.tasks:
        if not task.stardis_params:
            continue
        if obj_type == "camera" and task.stardis_params.camera_ref == old_name:
            task.stardis_params.camera_ref = new_name
        elif obj_type == "probe":
            refs = task.stardis_params.probe_refs
            task.stardis_params.probe_refs = [new_name if r == old_name else r for r in refs]
```

### 7.3 任务删除时

删除任务时检查是否有 HTPP 任务依赖（`InputFromTask`）：

```python
def _check_dependent_tasks(self, task_id: str) -> List[Task]:
    """返回依赖指定任务的所有 HTPP 任务。"""
    return [t for t in self._model.task_queue.tasks
            if isinstance(t.input_source, InputFromTask)
            and t.input_source.task_id == task_id]
```

若有依赖，提示用户：将依赖任务的 `input_source` 改为 `InputFromFile("")`（需手动指定）或一并删除。
