# EditorPreferences — 编辑器偏好设置

编辑器级别设置，与场景数据无关，持久化到项目根目录 `editor_settings.json`。

> **源文件**：`src/models/editor_preferences.py`、`src/panels/preferences_dialog.py`

---

## 1. 数据模型

```python
class StartupBehavior(Enum):
    NONE = "none"           # 无操作（空白场景）
    OPEN_LAST = "open_last" # 打开上次关闭的工程

@dataclass
class EditorPreferences:
    search_dirs: List[str]              # Stardis 搜索目录
    recent_exes: List[str]              # 已发现/使用过的 stardis.exe 路径（≤50）
    recent_work_dirs: List[str]         # 最近使用的工作目录（≤20）
    recent_projects: List[str]          # 最近工程文件路径（≤20）
    last_project_path: str              # 上次关闭时正在编辑的工程路径
    startup_behavior: StartupBehavior   # 启动行为
    exe_tags: Dict[str, str]            # 可执行文件标签（标签名 → 绝对路径）
```

### 1.1 可执行文件标签管理 (`exe_tags`)

`exe_tags` 统一管理 stardis 和 htpp 两类可执行文件，替代最初仅靠 `recent_exes` 平坦列表管理的方式：

```python
exe_tags: Dict[str, str] = field(default_factory=dict)
# key = 标签名 (用户自定义，如 "stardis-v3", "htpp-release")
# value = 绝对路径
```

**标签约定**：
- 标签名由用户自由定义，无强制前缀
- 建议命名习惯：`stardis-<版本/用途>`, `htpp-<版本/用途>`
- 自动发现（`scan_stardis_exes`）找到新 exe 时提示用户命名标签

**优先使用标签的好处**：
- 可执行文件路径集中管理，项目间共享
- 迁移机器时只需更新一处
- 避免项目文件中出现机器相关的绝对路径

**在 Task Runner 中的使用**：Task 的 `exe_ref` 字段引用标签名或绝对路径（含 `/`/`\` 判定为路径，否则判定为标签名），详见 `design/task_runner/01_数据模型.md` §3.1。

### 1.2 列表操作

- `add_recent_exe(path)`：新路径推入 `recent_exes` 头部，去重，截断到 50
- `add_recent_workdir(path)`：推入 `recent_work_dirs`，截断到 20
- `add_recent_project(path)`：推入 `recent_projects`，截断到 20

### 1.3 可执行文件自动搜索

`scan_stardis_exes()`：在 `search_dirs` 下递归搜索 `stardis.exe`，结果自动添加到 `recent_exes`。

## 2. PreferencesDialog

三标签页结构：

### 2.1 常规标签页

- 启动时默认行为：下拉选择「无操作（空白场景）」 / 「打开上次关闭的工程」

### 2.2 可执行文件标签页

统一管理 stardis + htpp 可执行文件的标签：

```
┌─ 可执行文件标签 ──────────────────────┐
│  标签名          │ 路径              │       │
│  stardis-v3     │ D:/tools/star...  │ [浏览] │
│  htpp-release   │ D:/tools/htpp...  │ [浏览] │
│  stardis-debug  │ D:/dev/stardis.. │ [浏览] │
│  ...            │                   │       │
│  [+ 添加]  [- 删除选中]  [扫描发现…]           │
└────────────────────────────────────────┘
```

- 「添加」：弹出两行输入（标签名 + 文件选择），选择可执行文件后添加到表格
- 「删除选中」：移除选中行。删除标签时检查当前项目是否有任务引用该标签，若有则提示警告
- 「扫描发现…」：在搜索目录下自动搜索 `stardis.exe`，对新发现的 exe 弹出命名对话框

**搜索目录**组保留于此标签页底部，用于配置自动发现的搜索路径：

```
┌─ 搜索目录 ────────────────────────────┐
│  D:/tools/                             │
│  C:/Program Files/Stardis/             │
│                                        │
│  [添加目录...]  [移除选中]              │
└────────────────────────────────────────┘
```

### 2.3 最近工程标签页

- 最近打开的工程文件列表（只读浏览）
- 清空列表

### 2.4 任务创建时的默认值

创建新任务时，自动填充：

| 字段 | 默认值来源 |
|------|-----------|
| `exe_ref` (stardis) | `prefs.exe_tags` 中第一个包含 "stardis" 的标签名，若无则空 |
| `exe_ref` (htpp) | `prefs.exe_tags` 中第一个包含 "htpp" 的标签名，若无则空 |
| `working_dir` | 场景文件所在目录 |
| `stardis_params.model_file` | `SceneWriter` 输出目录下的 `scene.txt` |
| `stardis_params.threads` | `os.cpu_count() or 4` |

## 3. 持久化

### 3.1 JSON 格式

```json
{
  "search_dirs": ["D:/tools/"],
  "recent_exes": ["D:/tools/stardis.exe"],
  "recent_work_dirs": ["D:/work/project1"],
  "recent_projects": ["D:/work/project1/scene.txt"],
  "last_project_path": "D:/work/project1/scene.txt",
  "startup_behavior": "open_last",
  "exe_tags": {
    "stardis-v3": "D:/tools/stardis-v3/stardis.exe",
    "stardis-debug": "D:/dev/stardis/build/stardis.exe",
    "htpp-release": "D:/tools/htpp.exe"
  }
}
```

### 3.2 向后兼容

- 加载不含 `exe_tags` 字段的旧设置文件时，使用默认空 dict
- 若旧文件存在 `recent_exes` 列表，可在迁移时为每个路径自动生成标签（以 exe 文件名为基础），或提示用户手动命名

### 3.3 v1 迁移

首次启动 v2 时，若 `editor_settings.json` 不存在但 `user_settings.json` 存在，自动迁移 `search_dirs`、`recent_exes`、`recent_work_dirs` 三个字段。

---

## 4. 关闭行为

窗口关闭时自动保存 `last_project_path`（当前正在编辑的文件路径）到偏好设置，供下次启动时恢复。
