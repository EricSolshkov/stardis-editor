# 场景管理与界面编辑系统

> **归档层次**：系统架构，面向全局理解与跨系统集成。
> 详细实现参见 [`design/scene_editor/`](../scene_editor/) 与 [`design/SceneTree_VTK_Interaction.md`](../SceneTree_VTK_Interaction.md)。

---

## 1. 系统定位

场景管理与界面编辑系统是 StardisEditor v2 的核心子系统，负责：

1. **场景数据建模**：以几何体（Body）为中心，组织表面区域（SurfaceZone）、边界条件、探针、摄像机、光源等场景实体
2. **场景文件 I/O**：解析/写出 Stardis `.txt` 场景文件及 `.stardis_project.json` 工程元数据
3. **三视图联动**：场景树（层级浏览）/ 属性面板（动态编辑）/ 3D 视口（VTK 渲染）三组件同步
4. **画笔标注**：在 3D 视口中通过涂选操作划定表面区域（BoundaryLabel 单元标注）

## 3. 数据模型层次

```
SceneModel
├── global_settings          (trad, scale)
├── bodies: List[Body]
│   └── Body
│       ├── name, stl_files
│       ├── volume: VolumeProperties     (type / material: MaterialRef / delta / ...)
│       └── surface_zones: List[SurfaceZone]
│           └── SurfaceZone
│               ├── zone_id, name
│               ├── source: ImportedSTL | PaintedRegion
│               └── boundary: TemperatureBC | ConvectionBC | FluxBC | CombinedBC
├── connections: List[Connection]
│   ├── SolidFluidConnection
│   └── SolidSolidConnection
├── probes: List[Probe]     (VolumeTemp / SurfaceTemp / SurfaceFlux)
├── cameras: List[IRCamera]
├── lights: List[SceneLight]  (DEFAULT / SPHERICAL_SOURCE / SPHERICAL_SOURCE_PROG)
└── ambient_intensity: float
```

### 3.1 边界条件类型映射

| 数据类 | 场景文件关键字 |
|-------|--------------|
| `TemperatureBC` | `T_BOUNDARY_FOR_SOLID` |
| `ConvectionBC` | `H_BOUNDARY_FOR_SOLID` |
| `FluxBC` | `F_BOUNDARY_FOR_SOLID` |
| `CombinedBC` | `HF_BOUNDARY_FOR_SOLID` |

## 4. 三组件通信协议

三组件通过 Qt 信号/槽解耦，**不直接互相引用**，所有消息经 `SceneEditor` 主窗口中转。

### 4.1 核心信号

| 发出方 | 信号 | 接收方动作 |
|--------|------|-----------|
| `SceneTreePanel` | `body_selected(name)` | Viewport 轮廓高亮 + Panel 显示体积属性 |
| `SceneTreePanel` | `zone_selected(bname, zname)` | Viewport 区域着色高亮 + Panel 显示边界参数 |
| `SceneViewport` | `body_picked(name)` | Tree 同步选中 + Panel 切换 |
| `SceneViewport` | `paint_changed(bname, zone_dict)` | SceneModel 更新 zone 数据 + Tree 刷新节点 |
| `SceneViewport` | `probe_placed(bname, x, y, z)` | SceneModel 添加探针 + Tree 创建节点 |
| `PropertyPanel` | `property_changed(name, key, value)` | SceneModel 写字段 + Viewport/Tree 按需刷新 |

### 4.2 3D 视口工作模式

| 模式 | 鼠标行为 | 触发条件 |
|------|---------|---------|
| **导航模式**（默认）| 中键旋转、滚轮缩放、左键拾取、双击放探针 | 工具栏切换 |
| **画笔模式** | 左键涂选、Ctrl+左键擦除、中键旋转 | 选中某 Body 后进入涂选编辑 |

## 5. 场景文件 I/O

### 5.1 解析流程（SceneParser）

```
.txt 文件
  → 按关键字行分组（SOLID / FLUID / T_BOUNDARY_FOR_SOLID / H_BOUNDARY_FOR_SOLID / ...）
  → 每类关键字构造对应 Body / SurfaceZone / Connection 对象
  → 填充 SceneModel
```

支持的关键字：`SOLID`、`FLUID`、`T_BOUNDARY_FOR_SOLID`、`H_BOUNDARY_FOR_SOLID`、`F_BOUNDARY_FOR_SOLID`、`HF_BOUNDARY_FOR_SOLID`、`SOLID_FLUID_CONNECTION`、`SOLID_SOLID_CONNECTION`、`SPHERICAL_SOURCE`、`SPHERICAL_SOURCE_PROG`

### 5.2 写出流程（SceneWriter）

```
SceneModel
  → scene.txt                     # Stardis 求解器直接消费
  → S_<body>.stl                  # Body 几何 STL
  → B_<zone>.stl                  # PaintedRegion 区域导出为 STL
  → .stardis_project.json         # 编辑器元数据（探针/摄像机/任务队列/zone_id 映射）
```

### 5.3 PaintedRegion 持久化策略

`PaintedRegion.cell_ids` 不直接写入 JSON；保存时导出 `B_*.stl`，加载时通过**三角形哈希匹配**（`triangle_hash_matcher`）从 STL 恢复 `cell_ids`，确保格式与 Stardis 兼容。

## 6. 画笔标注系统（SurfacePainter）

```
SceneViewport（画笔模式）
    │
    │ 鼠标拖拽 → 射线拾取命中的三角面
    ▼
SurfacePainter
    ├── vtkIntArray (BoundaryLabel)  — 每个三角面存储所属 zone_id
    ├── 撤销/重做栈（50 步）         — 命令模式，每次笔触为一条命令
    └── 多区域 LUT 着色              — 不同 zone_id 以不同颜色渲染
    │
    │ paint_changed 信号 → SceneModel 更新 PaintedRegion.cell_ids
    ▼
SceneModel（持久化到 B_*.stl）
```

## 7. VTK 高亮策略

| 选中对象 | 视觉效果 |
|---------|---------|
| Body | 黄色轮廓线（`vtkSilhouette`，线宽 3），跟随相机更新 |
| SurfaceZone | 轮廓线 + 区域原色显示，其余区域暗灰 (0.45, 0.45, 0.45) |
| Probe | 标记图标放大 + 坐标标注 |

---

## 8. 光源自动管理规则

- 无 DEFAULT / SPHERICAL_SOURCE 时，自动创建一个 DEFAULT 光源（`ensure_default_light()`）
- 添加 SPHERICAL_SOURCE 时，自动禁用所有 DEFAULT 光源
- 删除全部光源后，自动重建 DEFAULT 光源
- `ambient_intensity` 恒存在，通过 `actor.GetProperty().SetAmbient()` 施加，不受光源增删影响
