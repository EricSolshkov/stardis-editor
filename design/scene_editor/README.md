# Scene Editor v2 设计文档

本目录包含 Scene Editor v2 场景编辑子系统的设计文档，按**渐进式披露**原则分层组织——每层文档自成体系，读者可在需要时深入下一层，无需通读所有内容。

---

## 文档地图

| 文件 | 回答的问题 | 读者 | 改动频率 |
|------|-----------|------|---------|
| [01_概念与数据模型.md](01_概念与数据模型.md) | 系统是什么？数据怎么组织？文件怎么读写？ | 所有人（入门先读） | 低 |
| [02_UI行为规约.md](02_UI行为规约.md) | 用户能做什么？界面长什么样？ | 产品评审 / AI Agent 实现 UI | 中 |
| [03_组件通信协议.md](03_组件通信协议.md) | 三个面板之间如何传递消息？ | 实现者（编码前读） | 低 |
| [04_VTK实现方案.md](04_VTK实现方案.md) | 具体怎么用 VTK API 实现各功能？ | 仅实现者 | 高（随 VTK 版本变化）|
| [05_工程规划.md](05_工程规划.md) | 怎么分步实施？性能边界在哪里？ | 技术负责人 / Sprint 规划 | 低 |

## 推荐阅读路径

```
新成员入门：         01 → 02 → 03
实现某个功能：       01（了解数据）→ 02（明确行为）→ 03（接口信号）→ 04（编写代码）
评审 UI 设计：       只需读 01 + 02
排查组件通信问题：   直接跳 03
调试 VTK 渲染问题：  直接跳 04
```
