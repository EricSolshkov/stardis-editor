# 02 — UI 交互设计

本文档定义 Task Runner 在场景编辑器中的 UI 集成方案：场景树的任务节点、属性面板的任务编辑器、右键快捷创建、以及场景验证入口。

---

## 1. 场景树集成（A3 方案）

### 1.1 树结构

Tasks 作为顶层分组节点，与 Bodies / Probes / Cameras / Lights 同级：

```
SceneTree
├── [全局设置]
├── Bodies/
│   ├── Wall
│   └── Heater
├── Connections/
├── Probes/
│   ├── P1                  (VOLUME_TEMP)
│   └── P2                  (SURFACE_TEMP)
├── Cameras/
│   ├── Camera1
│   └── Camera2
├── Lights/
└── Tasks/                   ← 任务队列分组
    ├── [1] IR渲染-Camera1    (STARDIS / IR_RENDER)
    ├── [2] 后处理-Camera1    (HTPP / IMAGE, ← [1])
    └── [3] 探针计算-P1,P2   (STARDIS / PROBE_SOLVE)
```

### 1.2 节点类型与显示

| 节点 | NODE_TYPE 常量 | 显示文本 | 图标/装饰 |
|------|---------------|---------|----------|
| Tasks 分组 | `NODE_TASK_GRP` | "Tasks" | 文件夹图标 |
| 单个任务 | `NODE_TASK` | `"[序号] 名称"` | 根据状态着色 |

**任务节点着色规则**：

| 状态 | 颜色 | 含义 |
|------|------|------|
| 正常 | 默认 | 可运行 |
| 禁用 | 灰色 (0.5, 0.5, 0.5) | `enabled=False`，队列跳过 |
| 警告 | 橙色 (0.9, 0.6, 0.1) | 引用缺失（Camera/Probe 已删除）|
| 运行中 | 蓝色 (0.2, 0.6, 1.0) | 当前正在执行 |
| 完成 | 绿色 (0.2, 0.8, 0.2) | 执行成功 |
| 失败 | 红色 (0.9, 0.2, 0.2) | 执行失败 |

**任务节点文本格式**：`[序号] 名称 (类型简写)`

```
[1] IR渲染-Camera1       (Stardis/IR)
[2] 后处理-Camera1       (HTPP/Image ← [1])
[3] 探针计算-P1,P2       (Stardis/Probe)
```

### 1.3 数据角色

新增 `ROLE_TASK_ID` 存储 `task.id`，`ROLE_NODE_NAME` 存储 `task.name`。

### 2.2 TaskQueueEditor — 队列总览

选中 Tasks/ 分组节点时显示。

```
┌─────────────────────────────────────────┐
│  任务队列                        [▶ 运行全部]│
├─────────────────────────────────────────┤
│                                         │
│  ┌─ 错误处理策略 ──────────────────┐    │
│  │ 重试次数: [__0__]               │    │
│  │ 重试耗尽: (●) 取消队列          │    │
│  │           ( ) 跳过继续          │    │
│  │           ( ) 暂停等待          │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─ 环境变量（队列级）─────────────┐    │
│  │  变量名           │ 值          │    │
│  │  OMP_NUM_THREADS  │ 4           │    │
│  │  LD_LIBRARY_PATH  │ /opt/lib    │    │
│  │  ...              │             │    │
│  │  [+ 添加]  [- 删除选中]         │    │
│  └─────────────────────────────────┘    │
│  (队列环境变量作为所有任务的基准，       │
│   任务可在自身环境变量中覆盖或追加)      │
│                                         │
│  ┌─ 任务列表 ──────────────────────┐    │
│  │ ☑ [1] IR渲染-Camera1     [▲][▼]│    │
│  │ ☑ [2] 后处理-Camera1     [▲][▼]│    │
│  │ ☑ [3] 探针计算-P1,P2     [▲][▼]│    │
│  │                                 │    │
│  │ [+ 添加任务]  [🗑 删除选中]     │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─ 运行状态 ─────────────────────┐     │
│  │ (队列空闲 / 运行中 2/3 / 完成) │     │
│  │ [查看运行日志...]               │     │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

**交互说明**：

| 操作 | 效果 |
|------|------|
| `▶ 运行全部` | 触发队列执行（先保存场景 → 求值锁定 → 顺序执行） |
| `[▲][▼]` 按钮 | 调整任务顺序（依赖校验：移动后若违反约束，显示警告但不阻止） |
| `☑` 复选框 | 启用/禁用任务 |
| `+ 添加任务` | 弹出选择对话框：Stardis (Probe/Field/IR) / HTPP (Image/Map) |
| `🗑 删除选中` | 删除选中任务（若有其他任务依赖此任务，提示确认） |
| 双击任务行 | 跳转到该任务的编辑器（等同于在场景树中选中该任务） |

## 3. 右键菜单——快捷创建任务

### 3.1 Camera 节点右键

```
右键 Camera1 (NODE_CAMERA)
├── 删除
├── ──────────
├── 创建渲染任务…          → 自动创建一组:
│                            [Stardis IR_RENDER, camera_ref=Camera1]
│                            [HTPP IMAGE, input_source=FromTask(上面的)]
│                            两个任务追加到队列末尾
│                            焦点跳转到 Tasks/ 并选中第一个新任务
├── 创建渲染任务（仅计算）…  → 只创建 Stardis IR_RENDER 任务
└── 添加到现有任务 ▸        → 子菜单列出队列中已有的 IR_RENDER 任务
                              选中后将该任务的 camera_ref 替换为此 Camera
                              (仅当任务当前无相机引用或用户确认替换时)
```

### 3.2 Probe 节点右键

```
右键 P1 (NODE_PROBE)
├── 删除
├── ──────────
├── 创建探针计算任务…       → 创建 Stardis PROBE_SOLVE 任务
│                            probe_refs = [P1]
│                            追加到队列末尾
└── 添加到现有任务 ▸        → 子菜单列出队列中已有的 PROBE_SOLVE 任务
                              选中后将 P1 追加到该任务的 probe_refs
```

### 3.3 Tasks 分组节点右键

```
右键 Tasks/ (NODE_TASK_GRP)
├── 添加 Stardis 任务 ▸
│   ├── 探针求解
│   ├── 场求解
│   └── IR 渲染
├── 添加 HTPP 任务 ▸
│   ├── 图像模式
│   └── 映射模式
├── ──────────
├── 运行全部
└── 清空任务队列
```

### 3.4 单任务节点右键

```
右键 [1] IR渲染-Camera1 (NODE_TASK)
├── 运行此任务              → 单独执行（不走队列调度）
├── ──────────
├── 上移 / 下移
├── 启用 / 禁用
├── 复制任务                → 深拷贝追加到队列末尾（新 uuid）
├── ──────────
├── 删除
```

### 3.5 空白区域右键

在现有的"打开场景文件目录"之前追加：

```
右键 (空白)
├── 验证场景…               → 调用 stardis -d，读取输出展示
├── 打开场景文件目录
```

## 4. 信号定义

### 4.1 SceneTreePanel 新增信号

```python
# 任务管理
request_create_render_task    = pyqtSignal(str)          # camera_name → 创建 IR+HTPP 组
request_create_render_only    = pyqtSignal(str)          # camera_name → 仅创建 IR 任务
request_create_probe_task     = pyqtSignal(str)          # probe_name → 创建 PROBE_SOLVE
request_add_to_task           = pyqtSignal(str, str)     # (object_name, task_id) → 添加到现有任务
task_selected                 = pyqtSignal(str)          # task_id → 切换到任务编辑器
task_queue_selected           = pyqtSignal()             # 选中 Tasks 分组 → 队列总览
request_add_task              = pyqtSignal(str, str)     # (task_type, compute_mode) → 手动添加
request_delete_task           = pyqtSignal(str)          # task_id
request_move_task             = pyqtSignal(str, int)     # (task_id, direction: -1/+1)
request_run_task              = pyqtSignal(str)          # task_id → 单独运行
request_run_queue             = pyqtSignal()             # 运行全部
request_validate_scene        = pyqtSignal()             # 验证场景

# 引用级联（Camera/Probe 重命名时通知更新任务引用）
# → 由 SceneEditor 在处理重命名时直接调用 model 更新，不需额外信号
```

## 5. 场景验证入口

### 5.1 触发方式

| 入口 | 位置 |
|------|------|
| 场景树空白区域右键 → "验证场景…" | `_on_context_menu` 扩展 |
| 菜单栏 → 场景 → "验证场景" | 替换现有 stub |

### 5.2 执行流程

```
用户点击 "验证场景…"
  │
  ├── 1. 若场景未保存 → 提示先保存
  │
  ├── 2. 确定路径
  │      scene_file = 当前场景文件路径
  │      dump_file  = tempfile (或 scene 同目录下 .dump_model.txt)
  │
  ├── 3. 查找 stardis exe
  │      优先使用 EditorPreferences.exe_tags 中第一个 stardis 标签
  │      若无 → 提示用户选择
  │
  ├── 4. 执行
  │      QProcess: stardis -M scene.txt -d dump_model.txt
  │      等待完成（超时处理）
  │
  ├── 5. 读取 dump_model.txt 内容
  │
  └── 6. 弹出 QDialog 展示
         ├── QTextEdit (只读，等宽字体)
         ├── 内容: dump_model.txt 全文
         └── 按钮: [关闭] [复制内容]
```

## 7. 运行日志面板

任务执行时的 stdout/stderr 需要有地方展示。两种方案：

### 方案 A：任务编辑器内嵌日志

在 TaskEditor 底部添加可折叠的日志区域。优点：上下文集中。缺点：只看到当前选中任务的日志。

### 方案 B：独立日志对话框

点击"查看运行日志"弹出非模态对话框，包含所有任务的合并输出（按任务分 Tab 或带颜色前缀区分）。

**建议**：方案 B——独立日志对话框。队列运行时自动弹出。

```
┌─ 运行日志 ──────────────────────────────┐
│ [全部] [[1]IR渲染] [[2]后处理] [[3]探针]│ ← Tab 选择
│                                         │
│ [09:30:01] [1] 开始: stardis -M ...     │
│ [09:30:01] [1] Reading model...         │
│ [09:30:05] [1] Rendering: 50%...        │
│ [09:30:10] [1] 完成 (退出码: 0)         │
│ [09:30:10] [2] 开始: htpp -i ...        │
│ ...                                     │
│                                         │
│ [清空] [复制全部]              [关闭]    │
└─────────────────────────────────────────┘
```
