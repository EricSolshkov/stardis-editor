# 03 — 执行引擎

本文档定义 Task Runner 的运行时行为：命令行构建、进程管理、队列调度和错误处理。

---

## 1. 架构概览

```
SceneEditor
  │
  │ request_run_queue / request_run_task
  ▼
TaskRunner (QObject)               ← 核心调度器
  ├── validate_queue()             ← 运行前校验
  ├── resolve_all()                ← 引用求值 → ResolvedTask 列表
  ├── run_queue()                  ← 顺序执行队列
  │     ├── _run_next()
  │     │     ├── build_command()  ← 参数 → 命令行
  │     │     ├── QProcess.start() ← 启动子进程
  │     │     └── _on_finished()   ← 完成回调 → 成功/失败处理
  │     └── _handle_error()        ← 错误策略分发
  └── run_single(task_id)          ← 单任务执行（不走队列）
```

**职责边界**：

| 组件 | 职责 |
|------|------|
| `TaskRunner` | 队列调度、进程生命周期、错误处理 |
| `CommandBuilder` | 将 `ResolvedTask` 转为命令行参数列表 |
| `TaskEditor` / `TaskQueueEditor` | UI 交互，不涉及执行逻辑 |
| `SceneEditor` | 连接信号，协调保存/解引用/启动 |

### 2.3 HTPP 命令构建规则

#### 2.3.1 参数顺序与仅生成非默认值原则

htpp 命令行格式：`htpp [-fv] [-i image_options] [-m map_options] [-o output] [-t threads] [input]`

**核心规则**：仅当参数值**非默认**时才生成对应的标志/选项。这与 C 源码的解析逻辑一致（缺失的参数使用编译时默认值）。

**标志顺序**（严格遵守）：
1. `-f`（force）和 `-v`（verbose）
2. `-i image_options`（IMAGE 模式，可选）
3. `-m map_options`（MAP 模式，可选）
4. `-o output_file`（可选）
5. `-t threads_count`（可选）
6. Positional argument: input file

#### 2.3.2 IMAGE 模式：-i exposure[:white=...]

```python
def _build_image_options(params: HtppParams) -> Optional[str]:
    """
    生成 IMAGE 模式的 -i 参数值。
    空字符串表示全默认（不传 -i 标志），非空字符串表示参数组合。
    """
    parts = []
    
    # exposure：非默认时添加（默认 1.0）
    if params.exposure != 1.0:
        parts.append(f'exposure={params.exposure}')
    
    # white_scale：非 None 时添加（None 表示自动，对应 C 的 white=-1.0）
    if params.white_scale is not None:
        parts.append(f'white={params.white_scale}')
    
    if not parts:
        return None  # 无需 -i 标志
    return ':'.join(parts)
```

**示例表**

| HtppParams | 生成的 -i 参数 | 说明 |
|-----------|------------|------|
| `exposure=1.0, white_scale=None` | 无 | 全默认 |
| `exposure=2.0, white_scale=None` | `-i exposure=2.0` | 仅曝光非默认 |
| `exposure=1.0, white_scale=5000` | `-i white=5000` | 仅白点非默认 |
| `exposure=2.0, white_scale=5000` | `-i exposure=2.0:white=5000` | 两个都非默认 |

#### 2.3.3 MAP 模式：-m pixcpnt=X:palette=Y:range=A,B:gnuplot

```python
def _build_map_options(params: HtppParams) -> Optional[str]:
    """
    生成 MAP 模式的 -m 参数值。
    空字符串表示全默认（不传 -m 标志），非空字符串表示参数组合。
    """
    parts = []
    
    # pixel_component：非默认时添加（默认 0）
    if params.pixel_component != 0:
        parts.append(f'pixcpnt={params.pixel_component}')
    
    # palette：非空字符串时添加（空字符串 = "inferno" 默认）
    if params.palette:
        parts.append(f'palette={params.palette}')
    
    # range：仅当两个都非 None 时添加
    if params.range_min is not None and params.range_max is not None:
        parts.append(f'range={params.range_min},{params.range_max}')
    
    # gnuplot：bool 标志，True 时添加
    if params.gnuplot:
        parts.append('gnuplot')
    
    if not parts:
        return None  # 无需 -m 标志
    return ':'.join(parts)
```

**示例表**

| HtppParams | 生成的 -m 参数 | 说明 |
|-----------|------------|------|
| `pixcpnt=0, palette="", range=None, gnuplot=False` | 无 | 全默认 |
| `pixcpnt=3, palette="", range=None, gnuplot=False` | `-m pixcpnt=3` | 仅像素分量非默认 |
| `pixcpnt=0, palette="jet", range=None, gnuplot=False` | `-m palette=jet` | 仅调色板非默认 |
| `pixcpnt=3, palette="jet", range=0,100, gnuplot=True` | `-m pixcpnt=3:palette=jet:range=0,100:gnuplot` | 多个都非默认 |

#### 2.3.4 完整命令生成流程

```python
@staticmethod
def build_htpp(task: ResolvedTask) -> List[str]:
    """构建 htpp 命令行参数列表（不含 exe 路径）。"""
    args = []
    
    # 1. 标志
    if task.force_overwrite:
        args.append('-f')
    if task.verbose:
        args.append('-v')
    
    # 2. 模式参数（仅根据 htpp_mode 生成对应选项）
    if task.htpp_mode == HtppMode.IMAGE:
        img_opts = _build_image_options(task.htpp_params)
        if img_opts:  # 仅当非默认时添加
            args.extend(['-i', img_opts])
    elif task.htpp_mode == HtppMode.MAP:
        map_opts = _build_map_options(task.htpp_params)
        if map_opts:  # 仅当非默认时添加
            args.extend(['-m', map_opts])
    
    # 3. Positional 输入文件（必须）
    args.append(task.input_file)
    
    # 4. 输出文件（可选）
    if task.output_file:
        args.extend(['-o', task.output_file])
    
    # 5. 线程数（可选，非默认时添加）
    if task.threads != 4:  # C 源码默认 INT_MAX，Python 务实取 4
        args.extend(['-t', str(task.threads)])
    
    return args
```

**参数顺序验证**：将上述生成的 args 列表通过 `subprocess.list2cmdline()` 转为 shell 命令字符串时，应符合 `htpp [-fv] [-i] [-m] [-o] [-t] [input]` 的格式。

与 `StardisControlPanel` 保持一致的命名规则：

```python
def build_ir_output_filename(camera_snapshot) -> str:
    w, h = camera_snapshot.resolution
    spp = camera_snapshot.spp
    return f"IR_rendering_{w}x{h}x{spp}.ht"
```

此文件名用于：
- `task.output_redirect`（stardis stdout 重定向）
- 后续 HTPP 任务的 `input_file`（当 `InputFromTask` 时自动推导）

## 3. 引用求值与快照

### 3.1 ResolvedTask

```python
@dataclass
class ResolvedTask:
    """引用已解析、参数已锁定的任务快照。"""
    task: Task                              # 原始任务引用（用于错误报告）
    exe_path: str                           # 已解析的 exe 绝对路径（标签已查询为实际路径）
    working_dir: str                        # 已验证的工作目录
    env_vars: Dict[str, str]
    args: List[str]                         # 已构建的命令行参数
    output_file: Optional[str]              # 输出重定向绝对路径
    input_file: Optional[str]               # HTPP: 输入文件绝对路径
```

### 3.2 解析流程

```python
def resolve_all(queue: TaskQueue, model: SceneModel, prefs: EditorPreferences) -> List[ResolvedTask]:
    """
    遍历队列中所有 enabled=True 的任务，解引用并构建命令行。
    返回可直接执行的 ResolvedTask 列表。
    
    Raises: ValidationError 若校验失败。
    """
    resolved = []
    output_map = {}  # task_id → output_file_path（用于 HTPP 输入推导）
    queue_env = dict(queue.env_vars)  # 队列级环境变量副本

    for task in queue.tasks:
        if not task.enabled:
            continue

        # 0. 解析 exe_ref → 绝对路径
        exe_path = _resolve_exe_ref(task.exe_ref, prefs)

        # 1. 确定 working_dir（空则默认场景目录）
        working_dir = task.working_dir or model.scene_dir

        # 2. 解引用场景对象
        if task.task_type == TaskType.STARDIS:
            snapshot = _resolve_stardis(task, model)
        else:
            input_file = _resolve_htpp_input(task, output_map, working_dir)
            snapshot = _resolve_htpp(task, input_file)

        # 3. 合并环境变量：队列级 ← 任务级覆盖/追加
        merged_env = {**queue_env, **task.env_vars}

        # 4. 构建命令行
        if task.task_type == TaskType.STARDIS:
            args = CommandBuilder.build_stardis(snapshot)
        else:
            args = CommandBuilder.build_htpp(snapshot)

        # 5. 解析相对路径为绝对路径（输出重定向）
        output_file = None
        if task.output_redirect:
            output_file = os.path.join(working_dir, task.output_redirect)

        # 6. 记录输出（用于后续 HTPP 引用）
        rt = ResolvedTask(task=task, exe_path=exe_path,
                          working_dir=working_dir,
                          args=args, env_vars=merged_env,
                          output_file=output_file, ...)
        if task.compute_mode == ComputeMode.IR_RENDER:
            output_map[task.id] = rt.output_file
        resolved.append(rt)

    return resolved
```

### 3.3 exe_ref 解析

```python
def _resolve_exe_ref(exe_ref: str, prefs: EditorPreferences) -> str:
    """将 exe_ref（标签名或绝对路径）解析为已验证的绝对路径。"""
    if not exe_ref:
        raise ValidationError("可执行文件未指定")

    # 判定：含路径分隔符 → 绝对路径，否则 → 标签名
    if '/' in exe_ref or '\\' in exe_ref:
        # 绝对路径回退
        path = exe_ref
    else:
        # 标签查询
        path = prefs.exe_tags.get(exe_ref)
        if not path:
            raise ValidationError(f"找不到可执行文件标签 '{exe_ref}'，"
                                  "请在编辑器首选项中配置")

    if not os.path.isfile(path):
        raise ValidationError(f"可执行文件不存在: {path}")
    return path
```

### 3.7 HTPP 输入推导

```python
def _resolve_htpp_input(task: Task, output_map: dict, working_dir: str) -> str:
    """解析 HTPP 输入来源，返回绝对路径。"""
    src = task.input_source
    if isinstance(src, InputFromTask):
        if src.task_id not in output_map:
            raise ValidationError(
                f"HTPP 任务 '{task.name}' 引用的 Stardis 任务未找到或未产出 .ht 文件")
        return output_map[src.task_id]    # 已是绝对路径
    elif isinstance(src, InputFromFile):
        # file_path 存储为相对路径，解析为绝对路径
        abs_path = os.path.join(working_dir, src.file_path)
        if not os.path.isfile(abs_path):
            raise ValidationError(
                f"HTPP 输入文件不存在: {abs_path}")
        return abs_path
```

## 4. 队列调度器

### 4.1 TaskRunner

```python
class TaskRunner(QObject):
    """任务队列调度器，管理顺序执行和错误处理。"""

    # ── 信号 ──
    queue_started     = pyqtSignal()                    # 队列开始
    queue_finished    = pyqtSignal(bool)                # 队列结束 (all_success)
    task_started      = pyqtSignal(str)                 # task_id
    task_finished     = pyqtSignal(str, int)            # task_id, exit_code
    task_output       = pyqtSignal(str, str)            # task_id, text (stdout)
    task_error_output = pyqtSignal(str, str)            # task_id, text (stderr)
    task_skipped      = pyqtSignal(str, str)            # task_id, reason
    queue_paused      = pyqtSignal(str)                 # task_id (失败的任务)

    def __init__(self, parent=None):
        super().__init__(parent)
        self._process = QProcess(self)
        self._resolved: List[ResolvedTask] = []
        self._current_index: int = -1
        self._retry_remaining: int = 0
        self._error_policy: ErrorPolicy = ErrorPolicy()
        self._running: bool = False
        self._paused: bool = False

        self._process.readyReadStandardOutput.connect(self._on_stdout)
        self._process.readyReadStandardError.connect(self._on_stderr)
        self._process.finished.connect(self._on_process_finished)
```

### 4.2 执行流程

```python
def run_queue(self, resolved: List[ResolvedTask], policy: ErrorPolicy):
    """启动队列执行。"""
    self._resolved = resolved
    self._error_policy = policy
    self._current_index = -1
    self._running = True
    self._paused = False
    self.queue_started.emit()
    self._run_next()

def _run_next(self):
    """推进到下一个任务。"""
    self._current_index += 1
    if self._current_index >= len(self._resolved):
        self._running = False
        self.queue_finished.emit(True)
        return

    rt = self._resolved[self._current_index]
    self._retry_remaining = self._error_policy.retry_count
    self._start_task(rt)

def _start_task(self, rt: ResolvedTask):
    """启动单个任务的 QProcess。"""
    self.task_started.emit(rt.task.id)

    # 环境变量已在 resolve_all 中合并（队列级 + 任务级）
    if rt.env_vars:
        env = QProcessEnvironment.systemEnvironment()
        for k, v in rt.env_vars.items():
            env.insert(k, v)
        self._process.setProcessEnvironment(env)

    self._process.setWorkingDirectory(rt.working_dir)

    if rt.output_file:
        self._process.setStandardOutputFile(rt.output_file)

    self._process.start(rt.exe_path, rt.args)
```

### 4.3 完成回调与错误处理

```python
def _on_process_finished(self, exit_code, exit_status):
    rt = self._resolved[self._current_index]
    self.task_finished.emit(rt.task.id, exit_code)

    if exit_code == 0:
        # 成功 → 继续下一个
        self._run_next()
    else:
        self._handle_failure(rt, exit_code)

def _handle_failure(self, rt: ResolvedTask, exit_code: int):
    """根据错误策略处理失败。"""

    # 重试
    if self._retry_remaining > 0:
        self._retry_remaining -= 1
        self._start_task(rt)
        return

    # 重试耗尽 → 按策略处理
    action = self._error_policy.after_retries_exhausted

    if action == ErrorAction.CANCEL:
        self._cancel_remaining("任务失败，队列已取消")

    elif action == ErrorAction.SKIP:
        self._skip_current_and_dependents(rt)
        self._run_next()

    elif action == ErrorAction.PAUSE:
        self._paused = True
        self._running = False
        self.queue_paused.emit(rt.task.id)
        # 等待用户通过 resume_retry / resume_skip / cancel 恢复

def _cancel_remaining(self, reason: str):
    """取消未执行的所有任务。"""
    for i in range(self._current_index + 1, len(self._resolved)):
        self.task_skipped.emit(self._resolved[i].task.id, reason)
    self._running = False
    self.queue_finished.emit(False)

def _skip_current_and_dependents(self, failed_rt: ResolvedTask):
    """跳过当前任务及依赖它的 HTPP 任务。"""
    failed_id = failed_rt.task.id
    for i in range(self._current_index + 1, len(self._resolved)):
        rt = self._resolved[i]
        if (isinstance(rt.task.input_source, InputFromTask)
                and rt.task.input_source.task_id == failed_id):
            # 标记依赖任务也跳过
            self._resolved[i] = None  # 或标记 skip flag
            self.task_skipped.emit(rt.task.id,
                f"依赖的任务 '{failed_rt.task.name}' 已失败")
```

### 4.4 暂停恢复

```python
def resume_retry(self):
    """暂停后重试当前任务。"""
    if not self._paused:
        return
    self._paused = False
    self._running = True
    rt = self._resolved[self._current_index]
    self._start_task(rt)

def resume_skip(self):
    """暂停后跳过当前任务。"""
    if not self._paused:
        return
    self._paused = False
    self._running = True
    rt = self._resolved[self._current_index]
    self._skip_current_and_dependents(rt)
    self._run_next()

def cancel(self):
    """取消队列（暂停状态或运行状态均可调用）。"""
    if self._process.state() == QProcess.Running:
        self._process.kill()
        self._process.waitForFinished(3000)
    self._cancel_remaining("用户取消")
```

### 4.5 单任务执行

```python
def run_single(self, resolved: ResolvedTask):
    """独立执行单个任务（不走队列调度）。"""
    self._resolved = [resolved]
    self._error_policy = ErrorPolicy(retry_count=0, after_retries_exhausted=ErrorAction.CANCEL)
    self._current_index = -1
    self._running = True
    self.queue_started.emit()
    self._run_next()
```

## 5. 场景验证执行

场景验证不通过 TaskRunner，使用独立的轻量 QProcess：

```python
class SceneValidator(QObject):
    """stardis -d 场景验证。"""

    validation_finished = pyqtSignal(str, int)  # (output_text, exit_code)

    def validate(self, stardis_exe: str, scene_file: str, working_dir: str):
        dump_file = os.path.join(working_dir, ".scene_validation.txt")
        self._dump_file = dump_file
        self._process = QProcess(self)
        self._process.setWorkingDirectory(working_dir)
        self._process.finished.connect(self._on_finished)
        self._process.start(stardis_exe, ['-M', scene_file, '-d', dump_file])

    def _on_finished(self, exit_code, exit_status):
        text = ""
        if os.path.isfile(self._dump_file):
            with open(self._dump_file, 'r', encoding='utf-8', errors='replace') as f:
                text = f.read()
        self.validation_finished.emit(text, exit_code)
```

## 6. 运行前校验清单

`validate_queue()` 在 `resolve_all()` 之前调用，返回错误列表：

| 检查项 | 级别 | 描述 |
|--------|------|------|
| exe_ref 可解析 | ERROR | 标签名在 `prefs.exe_tags` 中存在，或绝对路径文件存在 |
| 工作目录存在 | ERROR | `os.path.isdir(task.working_dir)` |
| model_file 非空 | ERROR | stardis 任务必须指定模型文件 |
| Camera 引用有效 | ERROR | IR_RENDER 引用的相机必须在 model.cameras 中 |
