# 几何变换系统：完整设计方案

## 1. 概述

### 1.1 问题

Stardis 仿真引擎直接读取 STL 文件中的顶点坐标——即 STL 本身决定了几何体在场景中的位置、朝向和尺寸。当用户拥有一个标准零件模型（如单位立方体），想要将其放置到场景中的不同位置时，必须预先用外部工具（MeshLab、Blender）对 STL 做变换再导入——这极不方便。

### 1.2 目标

在 Stardis Editor 中内建「几何变换」功能：

1. 用场景目录下的 `raw_model/` 目录管理 **未经变换的原始 STL**
2. 在属性面板中为每个 Body 提供 **平移 / 旋转 / 缩放** 编辑控件
3. 3D 视口中 **实时预览** 变换后的几何体位置
4. 保存时自动将变换 **烘焙 (bake)** 到输出 STL，供 Stardis 直接使用
5. 从 `.txt` 文件直接加载的旧场景，变换显示为单位矩阵且 **不可编辑**

### 1.3 非目标

- 不实现骨骼动画、变形器等高级变换
- 不实现实例化（同一 raw STL 多次放置）——每个 Body 对应一个 STL
- 不修改 Stardis `.txt` 场景格式本身（变换信息纯粹存储在 `.stardis_project.json`）

## 2. 目录结构约定

### 2.1 场景目录布局

```
scene_dir/
├── raw_model/                 # 原始模型（用户管理，Editor 只读引用）
│   ├── S_FOAM.stl             # Body "FOAM" 的原始 STL
│   └── S_METAL.stl            # Body "METAL" 的原始 STL
│
├── scene.txt                  # Stardis 场景文件（引用变换后的 S_*.stl）
├── S_FOAM.stl                 # 【自动生成】变换后的 STL
├── S_METAL.stl                # 【自动生成】变换后的 STL
├── B_LAT.stl                  # 【自动生成】变换后的边界 STL
├── B_INTERIOR.stl             # 【自动生成】变换后的边界 STL
└── .stardis_project.json      # 编辑器元数据（含变换参数 + raw 路径）
```

### 2.2 规则

| 规则 | 说明 |
|------|------|
| `raw_model/` 为可选目录 | 不存在时回退到当前行为（直接引用 scene_dir 中的 STL） |
| Editor 不修改 `raw_model/` 内容 | 只读引用，用户用文件管理器或外部工具管理 |
| 输出 STL 写入 scene_dir 根目录 | 与现有行为一致（`S_*.stl`, `B_*.stl`） |
| `raw_model/` 中文件命名不强制 | Body 与 raw STL 的映射关系存储在 JSON，不依赖文件名约定 |

### 2.3 raw_model 发现逻辑

当打开一个场景时（`SceneParser.parse_file`），在完成 `.txt` 解析和 `.stardis_project.json` 加载后，执行 raw_model 发现：

```python
def _discover_raw_models(self, model: SceneModel, scene_dir: str):
    """如果 scene_dir/raw_model/ 存在，尝试关联 raw STL 到 Body。"""
    raw_dir = os.path.join(scene_dir, "raw_model")
    if not os.path.isdir(raw_dir):
        return

    for body in model.bodies:
        # 优先使用 JSON 中记录的 raw_stl_files
        if body.raw_stl_files:
            continue
        # 启发式匹配：raw_model/S_<body.name>.stl
        candidate = os.path.join(raw_dir, f"S_{body.name}.stl")
        if os.path.isfile(candidate):
            body.raw_stl_files = [candidate]
```

## 3. 数据模型

### 3.1 GeometryTransform

```python
import math
from dataclasses import dataclass, field
from typing import Tuple

@dataclass
class GeometryTransform:
    """
    几何体的 TRS 变换。

    应用顺序（右乘 / 局部坐标系）：Scale → Rotate → Translate
    即 v' = T · R · S · v

    旋转内部存储为单位四元数 (w, x, y, z)，UI 显示为欧拉角（度）。
    """
    translation: Tuple[float, float, float] = (0.0, 0.0, 0.0)
    rotation: Tuple[float, float, float, float] = (1.0, 0.0, 0.0, 0.0)  # (w, x, y, z)
    scale: Tuple[float, float, float] = (1.0, 1.0, 1.0)

    # ---------- 查询 ----------

    @property
    def is_identity(self) -> bool:
        """变换是否为单位变换。"""
        return (self.translation == (0.0, 0.0, 0.0) and
                self.rotation == (1.0, 0.0, 0.0, 0.0) and
                self.scale == (1.0, 1.0, 1.0))

    @property
    def has_negative_scale(self) -> bool:
        """缩放是否包含负值（镜像），会导致法线翻转。"""
        return any(s < 0 for s in self.scale)

    @property
    def determinant_sign(self) -> float:
        """变换行列式的符号。< 0 表示法线需要翻转。"""
        sx, sy, sz = self.scale
        return sx * sy * sz

    # ---------- 欧拉角转换（内旋 ZYX = 外旋 XYZ） ----------

    def to_euler_degrees(self) -> Tuple[float, float, float]:
        """
        四元数 → 欧拉角 (rx, ry, rz)（度），内旋 ZYX 约定。

        内旋 ZYX 等价于外旋 XYZ：
          1. 绕世界 X 轴旋转 rx
          2. 绕世界 Y 轴旋转 ry
          3. 绕世界 Z 轴旋转 rz
        """
        w, x, y, z = self.rotation

        # Roll (X)
        sinr_cosp = 2.0 * (w * x + y * z)
        cosr_cosp = 1.0 - 2.0 * (x * x + y * y)
        rx = math.atan2(sinr_cosp, cosr_cosp)

        # Pitch (Y) — clamp to avoid NaN near gimbal lock
        sinp = 2.0 * (w * y - z * x)
        sinp = max(-1.0, min(1.0, sinp))
        ry = math.asin(sinp)

        # Yaw (Z)
        siny_cosp = 2.0 * (w * z + x * y)
        cosy_cosp = 1.0 - 2.0 * (y * y + z * z)
        rz = math.atan2(siny_cosp, cosy_cosp)

        return (math.degrees(rx), math.degrees(ry), math.degrees(rz))

    def set_from_euler_degrees(self, rx: float, ry: float, rz: float):
        """
        欧拉角 (rx, ry, rz)（度）→ 四元数，内旋 ZYX 约定。
        """
        rx, ry, rz = math.radians(rx), math.radians(ry), math.radians(rz)
        cx, sx = math.cos(rx / 2), math.sin(rx / 2)
        cy, sy = math.cos(ry / 2), math.sin(ry / 2)
        cz, sz = math.cos(rz / 2), math.sin(rz / 2)

        self.rotation = (
            cx * cy * cz + sx * sy * sz,  # w
            sx * cy * cz - cx * sy * sz,  # x
            cx * sy * cz + sx * cy * sz,  # y
            cx * cy * sz - sx * sy * cz,  # z
        )

    # ---------- VTK 互操作 ----------

    def to_vtk_transform(self):
        """构建 vtkTransform 对象。"""
        import vtk
        t = vtk.vtkTransform()
        t.PostMultiply()
        # 应用顺序：S → R → T
        t.Scale(*self.scale)
        # 四元数→轴角
        w, x, y, z = self.rotation
        angle_rad = 2.0 * math.acos(max(-1.0, min(1.0, w)))
        if abs(angle_rad) < 1e-12:
            t.RotateWXYZ(0, 1, 0, 0)
        else:
            s = math.sin(angle_rad / 2.0)
            t.RotateWXYZ(math.degrees(angle_rad), x / s, y / s, z / s)
        t.Translate(*self.translation)
        return t

    def apply_to_polydata(self, polydata):
        """
        将变换烘焙到 vtkPolyData 的顶点上，返回新的 polydata。
        原始 polydata 不被修改。
        """
        import vtk
        transform_filter = vtk.vtkTransformPolyDataFilter()
        transform_filter.SetInputData(polydata)
        transform_filter.SetTransform(self.to_vtk_transform())
        transform_filter.Update()
        result = vtk.vtkPolyData()
        result.DeepCopy(transform_filter.GetOutput())
        # 负行列式→翻转法线
        if self.determinant_sign < 0:
            normals = vtk.vtkPolyDataNormals()
            normals.SetInputData(result)
            normals.FlipNormalsOn()
            normals.Update()
            result.DeepCopy(normals.GetOutput())
        return result

    # ---------- 序列化 ----------

    def to_dict(self) -> dict:
        d = {}
        if self.translation != (0.0, 0.0, 0.0):
            d["translation"] = list(self.translation)
        if self.rotation != (1.0, 0.0, 0.0, 0.0):
            d["rotation"] = list(self.rotation)
        if self.scale != (1.0, 1.0, 1.0):
            d["scale"] = list(self.scale)
        return d  # 单位变换 → 空字典

    @staticmethod
    def from_dict(d: dict) -> 'GeometryTransform':
        return GeometryTransform(
            translation=tuple(d.get("translation", [0.0, 0.0, 0.0])),
            rotation=tuple(d.get("rotation", [1.0, 0.0, 0.0, 0.0])),
            scale=tuple(d.get("scale", [1.0, 1.0, 1.0])),
        )
```

### 3.2 Body 字段扩展

```python
@dataclass
class Body:
    # ... 现有字段 ...
    stl_files: List[str] = field(default_factory=list)       # 当前引用的 STL 路径

    # ── 新增：几何变换 ──
    raw_stl_files: List[str] = field(default_factory=list)   # raw_model/ 中的原始 STL 路径
    transform: GeometryTransform = field(default_factory=GeometryTransform)
    transform_locked: bool = False   # True = 从 .txt 直接加载，不可编辑
```

**语义约定：**

| 场景来源 | `raw_stl_files` | `stl_files` | `transform` | `transform_locked` |
|----------|-----------------|-------------|-------------|---------------------|
| raw_model + JSON | `["raw_model/S_X.stl"]` | 同 raw_stl_files（视口加载用） | 用户设置 | `False` |
| 纯 .txt 加载 | `[]`（空） | `["S_X.stl"]`（场景目录中） | 单位变换 | `True` |
| 旧 JSON 无变换 | `[]` | `["S_X.stl"]` | 单位变换 | `False`（但是 raw_stl_files 为空时行为等同 locked） |

**关键不变量：**
- 视口始终加载 `stl_files[0]` 的 polydata（原始几何），并通过 Actor 的 UserTransform 显示变换
- `raw_stl_files` 非空时，保存时从 raw_stl_files 读取 → 烘焙 transform → 写入输出目录
- `raw_stl_files` 为空时，保存行为与现有逻辑一致（直接拷贝 stl_files）

## 4. 四元数与欧拉角约定

### 4.1 为什么使用四元数存储

| 问题 | 欧拉角 | 四元数 |
|------|--------|--------|
| 万向锁 (Gimbal Lock) | 存在（pitch=±90° 时丢失 1 自由度） | 无 |
| 插值 | 不自然（线性插值产生抖动） | Slerp 平滑 |
| 组合旋转 | 依赖矩阵，误差累积 | 直接四元数乘法 |
| 归一化 | 无意义 | 简单（除以模长） |

UI 使用欧拉角是因为直观易读——用户看到 `(0°, 45°, 0°)` 立刻理解含义；底层用四元数确保数学正确性。

### 4.2 约定

| 项目 | 约定 |
|------|------|
| 旋转顺序 | **内旋 ZYX**（等价于外旋 XYZ） |
| 四元数格式 | `(w, x, y, z)`，`w` 为标量部分 |
| 角度单位 | UI 显示度，内部弧度计算 |
| 正方向 | 右手系（VTK 默认） |
| 单位四元数 | `(1, 0, 0, 0)` = 无旋转 |

### 4.3 万向锁处理

当 pitch (ry) 接近 ±90° 时，`to_euler_degrees()` 的 roll/yaw 会发生退化。处理策略：

1. **四元数为唯一真值**——UI 显示的欧拉角仅为近似视图
2. UI 输入欧拉角时立即转为四元数存储
3. 不在连续帧间用欧拉角插值（当前场景不需要动画）
4. 若用户手动输入触发万向锁（pitch=±90°），显示警告 tooltip 但不阻止操作

## 5. VTK 视口集成

### 5.1 变换预览策略

使用 **Actor UserTransform**（不修改 polydata）：

```python
# SceneViewport._add_body_actor() 中：
def _add_body_actor(self, body: Body):
    # ... 现有逻辑：加载 STL → polydata → mapper → actor ...
    actor = vtk.vtkActor()
    actor.SetMapper(mapper)

    # ── 新增：应用变换 ──
    if not body.transform.is_identity:
        actor.SetUserTransform(body.transform.to_vtk_transform())

    self.renderer.AddActor(actor)
    self._body_actors[body.name] = actor
```

**关键优势：**

- polydata 中的 cell_ids 不变 → 画笔标注（BoundaryLabel）正常工作
- `vtkCellPicker` 返回的 cell_id 对应原始 polydata → 拾取无需转换
- 变换更新仅需 `actor.SetUserTransform(t)` + 渲染，无需重建 polydata

### 5.2 变换实时更新

```python
# SceneViewport 新增方法：
def update_body_transform(self, body_name: str, transform: GeometryTransform):
    """属性面板变换修改后调用，实时更新 Actor 位置。"""
    actor = self._body_actors.get(body_name)
    if not actor:
        return
    if transform.is_identity:
        actor.SetUserTransform(None)
    else:
        actor.SetUserTransform(transform.to_vtk_transform())
    self._render()
```

### 5.3 画笔兼容性

画笔系统 (`SurfacePainter`) 通过 cell_id 标注单元。使用 `actor.SetUserTransform()` 后：

- **拾取**：`vtkCellPicker` / `vtkHardwareSelector` 在渲染空间拾取，返回的 cell_id 仍然映射到原始 polydata → **无需修改画笔代码**
- **高亮显示**：单元高亮基于 BoundaryLabel 数组 → 随 Actor 变换正确渲染
- **撤销/重做**：操作的是 cell_id 集合 → 不受变换影响

```python
        self._tx.value(), self._ty.value(), self._tz.value())

    self.transform_changed.emit(self._body_name, t)
```

## 8. 场景保存（SceneWriter 修改）

### 8.1 Body STL 导出

```python
def save(self, model: SceneModel, output_dir: str, scene_filename: str = "scene.txt"):
    # ...
    body_rel = {}
    for body in model.bodies:
        rels = []
        # 确定源 STL 列表
        source_stls = body.raw_stl_files if body.raw_stl_files else body.stl_files

        for stl in source_stls:
            dst_name = os.path.basename(stl)
            # 移除可能的 raw_model 子目录前缀（输出到 scene_dir 根目录）
            dst = os.path.join(output_dir, dst_name)

            if body.raw_stl_files and not body.transform.is_identity:
                # 有 raw_model 且有非单位变换 → 烘焙变换
                raw_poly = self._load_stl_polydata(stl)
                transformed = body.transform.apply_to_polydata(raw_poly)
                self._write_stl(transformed, dst)
            else:
                # 无变换 或 无 raw_model → 直接拷贝
                self._copy_if_needed(stl, dst)

            rels.append(dst_name)
        body_rel[body.name] = rels
    # ... 后续边界 STL 处理和 scene.txt 生成 ...
```

### 8.2 边界 STL 导出

边界区域（SurfaceZone）的导出需要与 Body 的变换保持一致：

```python
for body in model.bodies:
    has_transform = (body.raw_stl_files and not body.transform.is_identity)

    for zone in body.surface_zones:
        dst_name = f"B_{zone.name}.stl"
        dst = os.path.join(output_dir, dst_name)

        if isinstance(zone.source, PaintedRegion) and zone.source.cell_ids:
            # 先从原始 polydata 提取子网格
            sub_poly = self._extract_submesh(body, zone)
            if has_transform:
                # 再应用变换
                sub_poly = body.transform.apply_to_polydata(sub_poly)
            self._write_stl(sub_poly, dst)
            zone_rel[(body.name, zone.name)] = [dst_name]

        elif isinstance(zone.source, ImportedSTL) and zone.source.stl_file:
            # ImportedSTL 只出现在 txt 加载的场景（transform_locked=True, 单位变换）
            # 直接拷贝即可
            self._copy_if_needed(zone.source.stl_file, dst)
            zone_rel[(body.name, zone.name)] = [dst_name]
```

### 8.3 VTK 辅助方法

```python
@staticmethod
def _load_stl_polydata(path: str):
    """加载 STL 文件为 vtkPolyData。"""
    import vtk
    reader = vtk.vtkSTLReader()
    reader.SetFileName(path)
    reader.Update()
    poly = vtk.vtkPolyData()
    poly.DeepCopy(reader.GetOutput())
    return poly

@staticmethod
def _write_stl(polydata, path: str):
    """将 vtkPolyData 写入 STL 文件（ASCII 格式）。"""
    import vtk
    writer = vtk.vtkSTLWriter()
    writer.SetFileName(path)
    writer.SetInputData(polydata)
    writer.SetFileTypeToASCII()
    writer.Write()
```

```json
      "raw_stl_files": ["raw_model/S_METAL.stl"],
      "transform": {}
    }
  }
}
```

**说明：**
- `raw_stl_files` 存储为**相对于 scene_dir 的路径**
- `transform` 为空字典 `{}` 时表示单位变换（`GeometryTransform.to_dict()` 返回 `{}`）
- 从 .txt 加载的场景（`transform_locked=True`）不写入 `body_transforms` 节——缺失即表示锁定

### 9.2 序列化（save_project 补丁）

```python
def save_project(self, path: str):
    data = { ... }  # 现有字段

    # ── 新增：变换数据 ──
    body_transforms = {}
    scene_dir = os.path.dirname(path)
    for body in self.bodies:
        if body.transform_locked:
            continue  # txt-loaded bodies 不序列化变换
        entry = {}
        # raw_stl_files → 相对路径
        if body.raw_stl_files:
            entry["raw_stl_files"] = [
                os.path.relpath(f, scene_dir) for f in body.raw_stl_files
            ]
        # transform → 仅非单位变换
        td = body.transform.to_dict()
        if td:
            entry["transform"] = td
        if entry:
            body_transforms[body.name] = entry

    if body_transforms:
        data["body_transforms"] = body_transforms

    with open(path, "w", encoding="utf-8") as f:
        json.dump(data, f, indent=2, ensure_ascii=False)
```

### 9.3 反序列化（load_project 补丁）

```python
def load_project(self, path: str):
    # ... 现有反序列化逻辑 ...

    # ── 新增：恢复变换数据 ──
    scene_dir = os.path.dirname(path)
    body_transforms = data.get("body_transforms", {})
    for body in self.bodies:
        bt = body_transforms.get(body.name)
        if bt:
            # 恢复 raw_stl_files（相对路径 → 绝对路径）
            raw_rels = bt.get("raw_stl_files", [])
            body.raw_stl_files = [
                os.path.normpath(os.path.join(scene_dir, r)) for r in raw_rels
            ]
            # 恢复 transform
            td = bt.get("transform", {})
            body.transform = GeometryTransform.from_dict(td)
            body.transform_locked = False
            # 如果有 raw_stl_files，更新 stl_files 指向 raw 以便视口加载原始几何
            if body.raw_stl_files:
                body.stl_files = list(body.raw_stl_files)
        else:
            # 没有变换记录 → 锁定
            body.transform = GeometryTransform()
            body.transform_locked = True
```

## 10. 场景解析：.txt 加载行为

从 `.txt` 文件直接加载场景时（无 `body_transforms` JSON 数据）:

1. `SceneParser.parse_file()` 正常解析 → Body 的 `stl_files` 指向 scene_dir 中的 STL
2. `load_project()` 尝试读 `body_transforms` → 空 → 所有 Body 的 `transform_locked = True`
3. `_discover_raw_models()` 检查 `raw_model/` 目录：
   - 如果存在且能匹配到 Body → 设置 `raw_stl_files` → 解锁 `transform_locked = False`
   - 不存在 → 保持锁定

**结果：**

| 有 raw_model/ + 有 JSON 变换 | 正常恢复变换，可编辑 |
|---|---|
| 有 raw_model/ + 无 JSON 变换 | 发现 raw_model，单位变换，可编辑 |
| 无 raw_model/ + 有 JSON 变换 | ⚠ raw 文件缺失，回退到 stl_files，变换数据保留但导出退化为拷贝 |
| 无 raw_model/ + 无 JSON 变换 | txt-only 场景，锁定只读 |

## 11. 边界区域处理

### 11.1 PaintedRegion（画笔标注）

- cell_ids 始终基于 **原始 polydata**（raw 或未变换的 STL）
- 画笔操作在原始 polydata 上进行（Actor 的 UserTransform 不影响 cell_id 映射）
- 导出时：从原始 polydata 提取子网格 → 应用 Body 的 transform → 写入 B_*.stl

### 11.2 ImportedSTL（导入的边界 STL）

- 仅出现在从 .txt 加载的场景中
- 此时 Body 的 `transform_locked = True`，变换为单位矩阵
- 导出时直接拷贝边界 STL，无需变换

### 11.3 一致性不变量

> **属于同一 Body 的所有几何输出（S_*.stl 和 B_*.stl）必须应用相同的变换。**

违反此不变量会导致 Stardis 出错（边界 STL 与主体 STL 不匹配）。代码中通过统一使用 `body.transform.apply_to_polydata()` 保证。

## 12. 法线处理

### 12.1 detect_normal_orientation 与变换

`detect_normal_orientation()` 基于符号体积判断法线朝向。变换中的缩放会影响判断：

| 缩放类型 | 行列式 det(S) | 法线影响 |
|----------|--------------|---------|
| 均匀正缩放 (k, k, k) | k³ > 0 | 无变化 |
| 非均匀正缩放 | > 0 | 无变化 |
| 含负值缩放 | < 0 | 法线翻转 |

**策略：**
1. 法线检测始终在 **原始 polydata** 上运行（与当前行为一致）
2. UI 缩放控件限定正值范围 `(0.001, 1000)` → 避免镜像
3. 若未来需要支持负缩放，在变换时检查 `determinant_sign` 并自动翻转法线（`apply_to_polydata` 已预留此逻辑）

```python
# tests/transform_tests/test_geometry_transform.py

class TestGeometryTransform:
    def test_identity(self):
        t = GeometryTransform()
        assert t.is_identity
        assert t.to_euler_degrees() == (0.0, 0.0, 0.0)

    def test_euler_roundtrip(self):
        """欧拉角 → 四元数 → 欧拉角 往返一致。"""
        for rx, ry, rz in [(30, 45, 60), (0, 90, 0), (-45, 0, 180)]:
            t = GeometryTransform()
            t.set_from_euler_degrees(rx, ry, rz)
            ex, ey, ez = t.to_euler_degrees()
            assert abs(ex - rx) < 0.01
            assert abs(ey - ry) < 0.01
            assert abs(ez - rz) < 0.01

    def test_quaternion_normalization(self):
        t = GeometryTransform()
        t.set_from_euler_degrees(30, 45, 60)
        w, x, y, z = t.rotation
        norm = math.sqrt(w*w + x*x + y*y + z*z)
        assert abs(norm - 1.0) < 1e-10

    def test_serialization_roundtrip(self):
        t = GeometryTransform(
            translation=(1.0, 2.0, 3.0),
            scale=(2.0, 2.0, 2.0))
        t.set_from_euler_degrees(0, 45, 0)
        d = t.to_dict()
        t2 = GeometryTransform.from_dict(d)
        assert t.translation == t2.translation
        assert t.scale == t2.scale
        # 四元数比较（允许符号翻转 q ≡ -q）
        for a, b in zip(t.rotation, t2.rotation):
            assert abs(a - b) < 1e-10

    def test_identity_serializes_empty(self):
        t = GeometryTransform()
        assert t.to_dict() == {}

    def test_vtk_transform_translation(self):
        """VTK 变换正确应用平移。"""
        t = GeometryTransform(translation=(10.0, 0.0, 0.0))
        vtk_t = t.to_vtk_transform()
        # 检查原点经过变换后的位置
        p = vtk_t.TransformPoint(0.0, 0.0, 0.0)
        assert abs(p[0] - 10.0) < 1e-10

    def test_apply_to_polydata(self):
        """烘焙变换到 polydata 上。"""
        import vtk
        # 创建一个简单的三角形
        points = vtk.vtkPoints()
        points.InsertNextPoint(0, 0, 0)
        points.InsertNextPoint(1, 0, 0)
        points.InsertNextPoint(0, 1, 0)
        tri = vtk.vtkTriangle()
        tri.GetPointIds().SetId(0, 0)
        tri.GetPointIds().SetId(1, 1)
        tri.GetPointIds().SetId(2, 2)
        cells = vtk.vtkCellArray()
        cells.InsertNextCell(tri)
        poly = vtk.vtkPolyData()
        poly.SetPoints(points)
        poly.SetPolys(cells)

        # 平移 (10, 0, 0)
        t = GeometryTransform(translation=(10.0, 0.0, 0.0))
        result = t.apply_to_polydata(poly)
        p0 = result.GetPoint(0)
        assert abs(p0[0] - 10.0) < 1e-10
```

## 15. 边界情况与降级策略

| 场景 | 处理 |
|------|------|
| raw_model/ 中 STL 被用户删除 | 保存时检查文件存在性 → 抛出警告对话框，保存中止 |
| raw_stl_files 路径包含中文 / 空格 | VTK STLReader 支持；使用 `os.path` 处理 |
| 超大 STL（>1M 面片） | `apply_to_polydata` 是 O(n) 操作，可接受；瓶颈仍在 STL 写入 |
| 变换后几何体超出 Stardis 坐标范围 | 不做限制——Stardis 自身会报错 |
| JSON 中 raw_stl_files 路径使用相对路径，场景目录移动后 | `os.path.join(scene_dir, rel)` 自然跟随，只要相对结构不变就正确 |
| 用户在画笔模式下修改变换 | 禁止——画笔模式激活时，变换 SpinBox 设为只读，避免 cell_id 语义冲突 |
