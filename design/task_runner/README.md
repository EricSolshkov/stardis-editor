## 版本历史

### v2.2.0
- **变量模板展开系统**（在 05_变量展开.md 中记录）
  - 任务输出文件名支持 `{VAR}` 模板语法，resolve_all() 阶段自动展开
  - 内置变量覆盖任务元数据、Stardis 参数、相机参数、HTPP 参数
  - `{env.XXX}` 引用环境变量，`{{`/`}}` 转义为字面量花括号
  - VariableLineEdit 控件提供 `{` 触发的上下文感知自动补全
  - EXE_TAG 变量自动处理路径→basename 转换，确保文件名安全
  - SPP 变量按计算模式区分数据源（IR_RENDER 用相机 spp，其他用 -n）
  - 向后兼容：无 Schema 变更，旧项目不受影响

### v2.1.0 (2026-03-10)
- **HTPP 参数完全对齐 C 源码**（在 01_数据模型.md §5 中记录）
  - HtppParams 精确映射 htpp C 实现（htpp 0.5）的参数结构与默认值
  - IMAGE 模式：exposure (default 1.0) / white_scale (None=auto or float=manual)
  - MAP 模式：pixel_component (0-7) / palette (48 预设，scmap 库全量) / range (auto/manual) / gnuplot (flag)
  - **仅生成非默认值原则**：命令行只传递参数值非默认的字段，与 C 源码解析逻辑一致
- **HTPP 命令行生成详化**（在 03_执行引擎.md §2.3 中记录）
  - 标志顺序：`[-fv] [-i] [-m] [-o] [-t] [input]`
  - IMAGE 模式：`-i exposure=X[:white=Y]`（仅非默认值）
  - MAP 模式：`-m pixcpnt=X[:palette=Y[:range=A,B[:gnuplot]]]`（仅非默认值）
  - 完整流程示意和参数组合表
- **HTPP UI 交互改进**（在 02_UI交互设计.md §2.3.5 中已有）
  - 像素分量和调色板改为预设下拉框（ComboBox with 8/48 options）
  - 白色缩放和范围支持自动/手动切换，实时联动禁用/启用对应输入框
  - 每个参数附带 tooltip 说明 C 源码行为和参数范围
- **日志和执行改进**
  - 任务启动时打印完整命令行日志（via `subprocess.list2cmdline()`）
  - stdout 重定向到文件时，避免过度处理；stderr 始终输出到 UI 日志
  - HTPP 输出文件路径转为绝对路径传给 `-o` 选项，避免权限问题

## 设计决策摘要

| 决策项 | 结论 |
|--------|------|
| UI 形式 | 场景树 Tasks/ 节点 + PropertyPanel 动态编辑器（A3 方案） |
| 参数格式 | 结构化 JSON，运行时校验并生成命令行 |
| 任务模式 | 一个 stardis 任务只能选择一种计算模式 |
| 引用策略 | 任务引用场景对象（Camera/Probe）名称，启动时求值锁定 |
| 持久化 | 绑定项目，存储在 `.stardis_project.json` |
| 任务组合 | 通用 `List[Task]` 建模，支持任务队列 |
| 右键快捷 | Camera → 自动创建 Stardis IR + HTPP 一组；Probe → 创建探针计算任务 |
| 错误处理 | 用户在队列节点配置：重试/暂停/取消，可级联 |
| 场景验证 | 扩展现有 stub，用 `stardis -d` 检查，读取输出展示 |
| 与旧面板关系 | 参考实现，不复用独立窗口 |
| **HTPP 参数** | **完全对齐 C 源码 `struct args`，包括参数默认值和生成规则** |
