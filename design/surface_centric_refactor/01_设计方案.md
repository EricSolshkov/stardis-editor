# 面中心化材质分配 — 设计方案

## 1. 动机

当前 BodyEditor 中的"法线朝向"下拉框（FRONT/BACK/BOTH）要求用户以"有 XX 物理属性 → 分配给某界面的某侧"的逆序思维操作。工程师编辑封闭腔体时的自然思维是：

```
看到一个界面 → 外侧是什么？内侧是什么？ → 分别分配腔体/材质
```

本次重构的目标：

1. **BodyEditor 中用"内侧/外侧"替代 FRONT/BACK/BOTH**
2. **自动检测封闭网格的法线朝向**（有符号体积法），减少手动确认
3. **对开放网格优雅降级为 FRONT/BACK**，用户可手动标注
4. **对 `.txt` 格式零侵入**，语义层仅存储在 `.stardis_project.json`

## 2. 数据模型变更

### 2.1 新增枚举与字段

```python
# scene_model.py 新增

class NormalOrientation(Enum):
    """法线朝向语义标注。"""
    UNKNOWN = "unknown"   # 未确认（开放网格或未检测）
    OUTWARD = "outward"   # 法线指向外侧
    INWARD  = "inward"    # 法线指向内侧
```

在 `Body` 中新增字段：

```python
@dataclass
class Body:
    name: str = ""
    stl_files: List[str] = field(default_factory=list)
    volume: VolumeProperties = field(default_factory=VolumeProperties)
    next_zone_id: int = 1
    surface_zones: List[SurfaceZone] = field(default_factory=list)
    normal_orientation: NormalOrientation = NormalOrientation.UNKNOWN  # 新增
```

### 2.2 语义映射规则

| normal_orientation | side=FRONT 对应 | side=BACK 对应 |
|---|---|---|
| OUTWARD | 外侧 | 内侧 |
| INWARD | 内侧 | 外侧 |
| UNKNOWN | FRONT（原始） | BACK（原始） |

`BOTH` 在语义层表示"内外两侧都是同一腔体"。

### 2.3 不变量

- `VolumeProperties.side` 字段保留，写 `.txt` 时仍使用 FRONT/BACK/BOTH
- `normal_orientation` 仅影响 UI 显示标签和项目 JSON 持久化
- 底层数据流不变：`Body.volume.side` 始终是 stardis 的真实值

## 3. 法线朝向自动检测

### 3.1 算法：有符号体积法

对封闭三角网格，使用散度定理计算有符号体积：

$$V_{signed} = \frac{1}{6} \sum_{i} \vec{v}_{0,i} \cdot (\vec{v}_{1,i} \times \vec{v}_{2,i})$$

- $V_{signed} > 0$ → 法线朝外 → `NormalOrientation.OUTWARD`
- $V_{signed} < 0$ → 法线朝内 → `NormalOrientation.INWARD`

### 3.2 开放网格检测

在计算有符号体积前，先判断网格是否封闭：

```python
feature_edges = vtk.vtkFeatureEdges()
feature_edges.BoundaryEdgesOn()
feature_edges.NonManifoldEdgesOn()
feature_edges.FeatureEdgesOff()
feature_edges.ManifoldEdgesOff()
feature_edges.SetInputData(poly)
feature_edges.Update()
is_closed = feature_edges.GetOutput().GetNumberOfCells() == 0
```

- 封闭 → 计算有符号体积 → 自动标注
- 不封闭 → `NormalOrientation.UNKNOWN`

### 3.3 调用时机

- `SceneParser.parse_file()` 完成后，对每个 Body 调用检测
- `SceneEditor._on_add_body()` 添加新几何体时调用检测
- 用户在 UI 中手动切换时覆盖自动结果

### 3.4 实现位置

在 `src/models/scene_model.py` 中新增工具函数：

```python
def detect_normal_orientation(poly: "vtk.vtkPolyData") -> NormalOrientation:
    """检测三角网格法线朝向。封闭网格返回 OUTWARD/INWARD，开放网格返回 UNKNOWN。"""
```

## 4. UI 变更

### 4.1 BodyEditor 修改

移除原有的 `side_combo`（FRONT/BACK/BOTH），替换为语义化控件：

```
┌─ 体积属性 ────────────────────────────────────┐
│  类型: [SOLID ▼]                               │
│  材质: [...▼] [管理...]                         │
│  λ: 0.04  ρ: 32  cp: 840                       │
│  ...                                            │
│                                                 │
│  ┌─ 法线与邻域 ──────────────────────────────┐  │
│  │  法线检测: ✅ 指向外侧 (自动检测)          │  │
│  │           [ ] 手动覆盖                     │  │
│  │                                            │  │
│  │  内侧/外侧: [外侧 ▼]                      │  │
│  │  含义: 此腔体位于法线指向一侧 (=FRONT)     │  │
│  └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

**具体控件**：

1. **法线状态标签** `normal_status_label`：显示自动检测结果
2. **手动覆盖复选框** `normal_override_check`：勾选后可手动选择
3. **手动选择下拉框** `normal_manual_combo`：OUTWARD / INWARD（仅 override 时启用）
4. **邻域选择下拉框** `side_semantic_combo`：替代原 side_combo
   - 当 orientation 已知时：显示"外侧" / "内侧" / "两侧"
   - 当 orientation 未知时：显示 "FRONT" / "BACK" / "BOTH"（降级）

### 4.2 邻域选择的映射逻辑

```python
def _semantic_to_side(self, semantic: str, orientation: NormalOrientation) -> Side:
    """将语义选项映射回 Side 枚举。"""
    if orientation == NormalOrientation.OUTWARD:
        return {"外侧": Side.FRONT, "内侧": Side.BACK, "两侧": Side.BOTH}[semantic]
    elif orientation == NormalOrientation.INWARD:
        return {"外侧": Side.BACK, "内侧": Side.FRONT, "两侧": Side.BOTH}[semantic]
    else:
        return {"FRONT": Side.FRONT, "BACK": Side.BACK, "BOTH": Side.BOTH}[semantic]

def _side_to_semantic(self, side: Side, orientation: NormalOrientation) -> str:
    """将 Side 枚举映射为语义显示。"""
    if orientation == NormalOrientation.OUTWARD:
        return {Side.FRONT: "外侧", Side.BACK: "内侧", Side.BOTH: "两侧"}[side]
    elif orientation == NormalOrientation.INWARD:
        return {Side.FRONT: "内侧", Side.BACK: "外侧", Side.BOTH: "两侧"}[side]
    else:
        return side.value  # "FRONT" / "BACK" / "BOTH"
```

### 4.3 交互行为

1. **加载 Body 时**：
   - 读取 `body.normal_orientation` 更新法线状态
   - 用映射函数将 `body.volume.side` 转为语义选项填入 combo

2. **用户改变邻域选项时**：
   - 用逆映射函数将语义选项转回 `Side` 写入 `body.volume.side`

3. **用户手动覆盖法线方向时**：
   - 更新 `body.normal_orientation`
   - 重新映射 combo 选项（标签变化但选中的底层 Side 不变）

## 5. 序列化变更

### 5.1 `.txt` 场景文件 — 无变更

SceneWriter 仍然从 `body.volume.side` 读取 FRONT/BACK/BOTH。

### 5.2 `.stardis_project.json` — 新增字段

```json
{
  "normal_orientations": {
    "FOAM": "outward",
    "AIR": "outward",
    "WALL": "unknown"
  },
  ...
}
```

### 5.3 加载逻辑

`SceneModel.load_project()` 中：

```python
# 恢复法线朝向标注
orientations = data.get("normal_orientations", {})
for body in self.bodies:
    val = orientations.get(body.name, "unknown")
    body.normal_orientation = NormalOrientation(val)
```

`SceneModel.save_project()` 中：

```python
# 保存法线朝向标注
data["normal_orientations"] = {
    body.name: body.normal_orientation.value
    for body in self.bodies
}
```

## 6. 加载流程（含自动检测）

```
scene.txt → SceneParser → SceneModel (side = FRONT/BACK/BOTH)
  │
  ├─ 有 .stardis_project.json？
  │   ├─ YES → load_project() 恢复 normal_orientation
  │   └─ NO  → 对每个 Body:
  │              ├─ 加载 STL → vtkPolyData
  │              ├─ 检测封闭性
  │              ├─ 封闭 → 计算有符号体积 → 设置 OUTWARD/INWARD
  │              └─ 开放 → 设置 UNKNOWN
  │
  └─ UI 按 normal_orientation 选择语义/降级显示
```
