## 1. 现状分析

### 当前 MaterialRef

```python
@dataclass
class MaterialRef:
    conductivity: float = 1.0    # λ (W/m/K)
    density: float = 1.0         # ρ (kg/m³)
    specific_heat: float = 1.0   # cp (J/kg/K)
```

**问题**：
- 每个 Body 持有独立的 `MaterialRef` 实例，无共享或复用机制
- 新建 Body 默认值 `(1.0, 1.0, 1.0)` 无物理意义，用户必须手动逐一修改
- 多个几何体使用同一材质时需重复输入，容易出错
- 无法快速查阅常用工程材质参数

### 设计约束

- **Stardis 场景文件格式不变**：`.txt` 文件中 λ/ρ/cp 始终以内联数值写出
- **材质名称是编辑器概念**：Stardis 求解器不识别材质名，仅使用数值
- **向后兼容**：已有项目文件和场景文件无需任何修改即可正常加载

## 2. 数据模型

### 2.1 Material — 命名材质

```python
@dataclass
class Material:
    """可复用的命名物理材质。"""
    name: str                        # 唯一标识名（如 "Copper", "Air"）
    conductivity: float              # λ (W/m/K)
    density: float                   # ρ (kg/m³)
    specific_heat: float             # cp (J/kg/K)
    category: str = ""               # 分类标签（"金属", "流体", "绝缘体", "自定义"）
    description: str = ""            # 可选备注
    is_builtin: bool = False         # True → 内置材质，用户不可删除/修改名称
```

**字段说明**：
- `name` 在库中唯一，用作查找键
- `category` 用于 UI 分组过滤，不影响逻辑
- `is_builtin` 标记内置材质，UI 中显示为只读（用户可复制后另存为自定义材质）

### 2.2 MaterialDatabase — 材质库管理器

```python
@dataclass
class MaterialDatabase:
    """材质数据库，管理内置 + 用户自定义材质。"""
    _materials: Dict[str, Material] = field(default_factory=dict)

    # ── 查询 ──

    def get(self, name: str) -> Optional[Material]:
        """按名称查找。"""

    def list_all(self) -> List[Material]:
        """返回全部材质，按 category → name 排序。"""

    def list_by_category(self, category: str) -> List[Material]:
        """按分类过滤。"""

    def categories(self) -> List[str]:
        """返回所有分类标签（去重、排序）。"""

    def contains(self, name: str) -> bool:
        """名称是否已存在。"""

    # ── 修改 ──

    def add(self, material: Material) -> bool:
        """添加材质。名称冲突时返回 False。"""

    def update(self, name: str, material: Material) -> bool:
        """更新已有材质。内置材质只允许修改 description。"""

    def remove(self, name: str) -> bool:
        """删除材质。内置材质不可删除。"""

    def duplicate(self, name: str, new_name: str) -> Optional[Material]:
        """复制材质并以新名称存入。"""

    # ── 持久化 ──

    def save(self, path: str):
        """保存用户自定义材质到 JSON 文件（不保存内置材质）。"""

    def load(self, path: str):
        """从 JSON 文件加载用户自定义材质，与内置材质合并。"""

    # ── 工厂方法 ──

    @staticmethod
    def create_default() -> 'MaterialDatabase':
        """创建包含全部内置材质的数据库。"""
```

### 2.3 MaterialRef 扩展

`MaterialRef` 增加可选字段以记录来源：

```python
@dataclass
class MaterialRef:
    conductivity: float = 1.0
    density: float = 1.0
    specific_heat: float = 1.0
    source_material: str = ""    # 新增：来源材质名称（空串 = 手动输入）
```

**语义**：
- `source_material` 仅为 UI 提示用途（显示"来自 Copper"），**不是活引用**
- 应用材质时执行**值拷贝**：将 Material 的 λ/ρ/cp 复制到 MaterialRef
- 用户后续手动修改 λ/ρ/cp 时，自动清空 `source_material`（表示已脱离模板）
- 序列化到 `.stardis_project.json`，不影响场景 `.txt` 文件

**为什么不用活引用**：
- 场景 `.txt` 写出时必须内联数值，活引用无法直接序列化
- 修改材质库不应悄悄改变已有场景的物理参数
- 值拷贝语义简单明确，降低 bug 风险

## 4. 持久化方案

### 4.1 存储位置

| 数据 | 存储路径 | 说明 |
|------|----------|------|
| 内置材质 | 源码硬编码 | `_BUILTIN_MATERIALS` 常量列表 |
| 用户自定义材质 | `material_database.json`（项目根目录） | 编辑器级别，与 `editor_settings.json` 同级 |
| Body 的材质来源标记 | `.stardis_project.json` | 已有 `body_zone_ids` 旁新增 `body_materials` |

### 4.2 material_database.json 格式

```json
{
  "version": 1,
  "materials": [
    {
      "name": "MyAlloy",
      "conductivity": 45.0,
      "density": 7200.0,
      "specific_heat": 460.0,
      "category": "自定义",
      "description": "项目专用合金"
    }
  ]
}
```

**规则**：
- 仅保存用户自定义材质（`is_builtin=False`）
- 加载时与内置材质合并；用户材质名不可与内置材质同名
- `version` 字段支持未来格式迁移

### 4.3 .stardis_project.json 扩展

在现有 `save_project` / `load_project` 中新增 `body_materials` 段：

```json
{
  "probes": [...],
  "cameras": [...],
  "body_zone_ids": {...},
  "body_materials": {
    "FOAM": "Polyurethane_Foam",
    "SHELL": "Steel_Stainless",
    "AIR": ""
  }
}
```

**读取逻辑**：
- 加载项目时，若 `body_materials` 存在且对应材质名在数据库中，仅用于 UI 显示来源标签
- 不覆盖 Body 的实际 λ/ρ/cp 值（值来自场景 `.txt` 的解析结果）
- 旧项目文件无 `body_materials` 字段 → 所有 Body 的 `source_material` 为空串，完全向后兼容

## 5. 初始化与生命周期

```
程序启动
  │
  ├── MaterialDatabase.create_default()     # 加载内置材质
  │
  ├── database.load("material_database.json")  # 合并用户自定义材质
  │
  └── SceneEditor 持有 self._material_db 实例
          │
          ├── 打开场景 → 解析 Body → 从 project.json 恢复 source_material
          │
          ├── BodyEditor 引用 material_db 提供选择器
          │
          ├── MaterialManagerDialog 引用 material_db 提供 CRUD
          │
          └── 退出时 → database.save("material_database.json")
```

## 6. 与场景编辑器的交互协议

### 6.1 材质应用（Apply Material）

```
用户在 BodyEditor 选择材质 "Copper"
  │
  ├── body.volume.material.conductivity = 400
  ├── body.volume.material.density = 8960
  ├── body.volume.material.specific_heat = 385
  ├── body.volume.material.source_material = "Copper"
  │
  ├── BodyEditor spinbox 更新显示
  │
  └── property_changed 信号 → SceneEditor 标记 dirty
```

### 6.2 手动修改覆盖

```
用户手动修改 λ spinbox (400 → 410)
  │
  ├── body.volume.material.conductivity = 410
  ├── body.volume.material.source_material = ""     # 清空来源标记
  │
  └── BodyEditor 材质选择器显示 "自定义"
```

### 6.3 从 Body 保存为材质

```
用户右键 Body → "保存当前材质到库"
  │
  ├── 弹出对话框：输入名称、选择分类
  │
  ├── MaterialDatabase.add(Material(name=..., λ=..., ρ=..., cp=...))
  │
  └── 成功 → 刷新材质选择器列表
```

### 6.4 信号流

```
MaterialDatabase
  │  material_added(name: str)        ← add / duplicate
  │  material_removed(name: str)      ← remove
  │  material_updated(name: str)      ← update
  │
  └───▶ BodyEditor 刷新材质下拉列表
  └───▶ MaterialManagerDialog 刷新列表
```

> **实现选择**：MaterialDatabase 可继承 QObject 以发射信号，
> 或使用回调模式在 SceneEditor 中协调更新（视实现复杂度而定）。
