# 05 — 变量模板展开

本文档定义任务字符串字段中的 `{VAR}` 模板变量语法、展开引擎实现、UI 自动补全交互及集成点。

---

## 1. 概述

任务的输出文件名等字符串字段支持 **模板变量**，在执行前自动展开为任务上下文中的实际值。用户可以编写如 `render_{CAMERA}_{SPP}spp` 的模板，系统在 `resolve_all()` 阶段展开为 `render_cam1_1000000spp`。

**设计目标**：

- 消除手动命名中的参数遗漏和不一致
- 批量任务时自动区分输出文件（SPP/相机/调色板等维度）
- 零侵入：模板串是合法字符串值，不改变序列化格式
- 向后兼容：不含 `{VAR}` 的字符串原样通过

---

## 2. 语法

| 语法 | 含义 | 示例 |
|------|------|------|
| `{VAR_NAME}` | 替换为变量值 | `{SPP}` → `1000000` |
| `{env.XXX}` | 替换为环境变量 XXX 的值 | `{env.SCENE}` → `city` |
| `{{` | 字面量 `{` | `{{text}}` → `{text}` |
| `}}` | 字面量 `}` | 同上 |

**变量名规则**：`\w+` 或 `\w+\.\w+`（仅用于 `env.` 前缀）。

**匹配正则**：`r'\{(\w+(?:\.\w+)?)\}'`

---

## 3. 可用变量

### 3.1 变量清单

| 分类 | 变量名 | 来源 | 说明 | 适用任务类型 |
|------|--------|------|------|-------------|
| **元数据** | `TASK_NAME` | `task.name` | 任务名称 | 全部 |
| | `TASK_INDEX` | 队列位置 | 1-based 序号 | 全部 |
| | `EXE_TAG` | `task.exe_ref` | 可执行文件标签名（见 §3.2） | 全部 |
| **Stardis** | `SPP` | 见 §3.3 | 采样数 | STARDIS |
| | `THREADS` | `sp.threads` | 线程数 (`-t`) | STARDIS / HTPP |
| | `VERBOSITY` | `sp.verbosity` | 详细度 (`-V`) | STARDIS |
| | `MODEL` | `sp.model_file` | 模型文件名（无扩展名） | STARDIS |
| | `ALGO` | `sp.advanced.diff_algorithm` | 传导步算法 (`-a`)，默认 `dsphere` | STARDIS |
| | `PICARD` | `sp.advanced.picard_order` | Picard 阶数 (`-o`) | STARDIS |
| **相机** | `CAMERA` | `sp.camera_ref` | 相机名称 | STARDIS (IR_RENDER) |
| | `WIDTH` | `resolved_camera["resolution"][0]` | 图像宽度（像素） | STARDIS (IR_RENDER) |
| | `HEIGHT` | `resolved_camera["resolution"][1]` | 图像高度（像素） | STARDIS (IR_RENDER) |
| | `FOV` | `resolved_camera["fov"]` | 视场角（度） | STARDIS (IR_RENDER) |
| **HTPP** | `PALETTE` | `hp.palette` | 调色板名称，默认 `inferno` | HTPP |
| | `PIXCPNT` | `hp.pixel_component` | 像素分量 (0-7) | HTPP |
| | `EXPOSURE` | `hp.exposure` | 曝光度 | HTPP |
| | `INPUT` | 推导的输入文件 | 输入文件名（无扩展名） | HTPP |
| **环境** | `env.XXX` | 合并后环境变量 | 队列级 + 任务级环境变量 | 全部 |

### 3.2 EXE_TAG 的路径安全处理

`exe_ref` 既可以是标签名（如 `stardis_v3`）也可以是绝对路径（如 `C:\tools\stardis.exe`）。直接将路径插入文件名会导致非法字符（`\`、`:`）。

**规则**：

```python
if '/' in ref or '\\' in ref:
    # 绝对路径 → basename 无扩展名
    EXE_TAG = os.path.splitext(os.path.basename(ref))[0]
else:
    # 标签名 → 原样使用
    EXE_TAG = ref
```

| `exe_ref` | `EXE_TAG` |
|-----------|-----------|
| `stardis_v3` | `stardis_v3` |
| `C:\tools\stardis.exe` | `stardis` |
| `/usr/local/bin/stardis` | `stardis` |

### 3.3 SPP 的数据源区分

`SPP` 根据计算模式从不同来源取值：

| 计算模式 | SPP 来源 | 原因 |
|---------|---------|------|
| `IR_RENDER` | `resolved_camera["spp"]` | `-R` 模式的采样数由相机参数指定，不使用 `-n` |
| `PROBE_SOLVE` | `sp.samples` | `-n` 参数 |
| `FIELD_SOLVE` | `sp.samples` | `-n` 参数 |

## 4. 适用字段

当前支持变量展开的字段：

| 所属 | 字段 | UI 控件 |
|------|------|---------|
| `Task` | `output_redirect` | stdout 重定向文件路径 |
| `Task` | `stderr_redirect` | stderr 重定向文件路径 |
| `HtppParams` | `output_file` | HTPP 输出文件路径 (`-o`) |

这些字段在 `resolve_all()` 的第 4 步被展开（见 §5）。

## 5. 展开引擎

### 5.1 模块结构

```
src/task_runner/variable_expander.py
  ├── VariableError              ← 展开失败异常
  ├── build_variable_registry()  ← 收集上下文 → {name: value} 字典
  ├── inject_input_variable()    ← 追加 INPUT 变量（HTPP 推导后）
  ├── expand_variables()         ← 模板替换引擎
  └── list_available_variables() ← UI 自动补全用变量列表
```

### 5.2 展开算法

```python
def expand_variables(template: str, registry: Dict[str, str]) -> str:
    # 1. 转义处理：{{ → 占位符，}} → 占位符
    # 2. 正则匹配 {VAR}，从 registry 查找替换
    #    未找到 → 抛出 VariableError（列出可用变量）
    # 3. 还原占位符 → 字面量 { }
```

### 5.3 在 resolve_all() 中的集成位置

```
resolve_all(task_queue) 执行流程：
  ┌──────────────────────────────────────────────────┐
  │ Step 1: 基础校验（exe_ref 解析、model_file 检查）  │
  │ Step 2: 引用求值（Camera/Probe → 冻结快照）       │
  │ Step 3: 构建变量注册表                             │  ← build_variable_registry()
  │ Step 4: 展开模板变量                               │  ← expand_variables()
  │         · output_redirect                         │
  │         · stderr_redirect                         │
  │         · htpp_params.output_file                  │
  │ Step 5: 构建命令行（展开后的值参与 -o 等参数）      │  ← build_htpp() / build_stardis()
  └──────────────────────────────────────────────────┘
```

**关键顺序**：变量展开在引用求值之后（需要 `resolved_camera` 提供 SPP/WIDTH 等值），在命令行构建之前（展开后的 `output_file` 要传给 `-o`）。

### 5.4 错误处理

| 场景 | 行为 |
|------|------|
| 模板中引用不存在的变量 | `VariableError`，消息包含可用变量列表 |
| `VariableError` 在 `resolve_all()` 中 | 被捕获并包装为 `ValidationError`，阻止任务执行 |
| 模板为空或不含 `{VAR}` | 原样返回，不报错 |
| `{{` / `}}` 转义 | 正常还原为字面量 `{` / `}` |

## 6. UI 自动补全

### 6.1 VariableLineEdit 控件

`VariableLineEdit(QLineEdit)` 替代原有的 `QLineEdit`，提供 `{` 触发的变量弹出补全。

**交互流程**：

1. 用户在输入框键入 `{` → 出现候选变量下拉列表
2. 继续输入字母可过滤候选（如 `{SP` → 仅显示 `SPP`）
3. 选择候选项 → 自动补全为 `{VAR_NAME}`（含闭合花括号）
4. Escape 或失焦 → 关闭弹出框

**键盘行为**：

| 按键 | 弹出框打开时 | 弹出框关闭时 |
|------|-------------|-------------|
| `↑` / `↓` | 导航候选列表 | 默认行为 |
| `Enter` / `Return` | 插入选中项 | 默认行为 |
| `Escape` | 关闭弹出框 | 默认行为 |

### 6.2 变量列表的上下文感知

`list_available_variables(task_type, compute_mode, htpp_mode)` 根据任务类型返回不同的变量子集：

- **STARDIS**：元数据 + Stardis 参数 + （IR_RENDER 时追加相机参数）
- **HTPP**：元数据 + HTPP 参数（THREADS/PALETTE/PIXCPNT/EXPOSURE/INPUT）
- 末尾始终附加 `env.XXX` 提示

### 6.3 命令预览

`TaskEditor` 在属性面板底部显示命令预览，当任务参数变化时实时更新。模板变量以原始 `{VAR}` 形式显示（不在预览中展开，因为此时尚无执行上下文）。

---

## 7. 向后兼容性

- **无 Schema 变更**：模板字符串是合法的字符串值，`.stardis_project.json` 格式不变
- **旧项目**：不含 `{VAR}` 的字符串在 `expand_variables()` 中原样通过
- **新项目在旧版本**：`{VAR}` 未被展开，直接作为字面量传入命令行（功能降级但不崩溃）

---

## 8. 使用示例

### 8.1 IR 渲染输出命名

```
模板:  render_{CAMERA}_{SPP}spp_{WIDTH}x{HEIGHT}
上下文: camera_ref="cam_front", spp=1000000, resolution=(1920, 1080)
结果:  render_cam_front_1000000spp_1920x1080
```

### 8.2 HTPP 映射输出

```
模板:  map_{INPUT}_{PALETTE}_c{PIXCPNT}
上下文: input_file="render_cam1.bin", palette="inferno", pixel_component=3
结果:  map_render_cam1_inferno_c3
```

### 8.3 批量任务区分

```
模板:  output_{EXE_TAG}_{TASK_INDEX}
上下文: exe_ref="stardis_v3", task_index=2
结果:  output_stardis_v3_2
```

### 8.4 环境变量

```
模板:  {env.PROJECT}_result
上下文: merged_env={"PROJECT": "city_ir"}
结果:  city_ir_result
```

### 8.5 转义

```
模板:  log_{{debug}}_{TASK_NAME}
结果:  log_{debug}_mytask
```
