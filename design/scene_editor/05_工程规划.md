# Scene Editor v2：工程规划

> **阅读层次**：项目管理层，技术负责人 / Sprint 规划使用。
> 本文档包含：增量实施步骤、性能边界与优化策略。

---

## 1. 实施分步

本功能系统建议按以下顺序增量实现，每步均可独立交付验证：

| 步骤 | 内容 | 交付验证 |
|------|------|---------|
| **S1** | SceneModel 数据模型（几何中心版） | 单元测试：创建 / 序列化 / 反序列化 |
| **S2** | 场景树面板（只读） | 从 SceneModel 渲染出正确的树结构 |
| **S3** | 多对象 VTK 渲染 + 按 BoundaryLabel 着色 | 加载 porous 场景，表面按区域着色 |
| **S4** | 场景树 ↔ 视口双向选择联动 | 点树→高亮3D，点3D→树同步 |
| **S5** | 属性面板动态切换 | 选中不同对象类型，面板内容正确切换 |
| **S6** | 涂选模式：画刷 + 撤销/重做 | 屏幕空间画刷涂色 + Ctrl 擦除 + Ctrl+Z 撤销 |
| **S7** | 涂选区域导出为 STL + 场景保存 | 涂选 → 保存 → B_*.stl 导出 + zone_parent_map 写入 JSON → stardis 能正确读取 |
| **S8** | 从 .txt 导入 + 三角形哈希匹配 | 导入现有场景 → 从 JSON 读父子关系（或弹窗选择）→ 哈希匹配恢复涂选 → 正确显示几何中心树结构 |
| **S9** | 覆盖率检查 + 视觉提示 | 未覆盖区域灰色显示，覆盖率数值正确 |
| **S10** | 探针系统：双击放置 + 拖拽 + 属性面板 | 双击几何体放置探针，拖拽移动，属性面板编辑参数 |
| **S11** | 探针与任务面板联动 | 探针列表自动填充到求解任务的 -p/-P/-f 参数 |

## 2. 性能边界与优化策略

### 2.1 大模型优化

| 三角面规模 | 措施 |
|-----------|------|
| < 10K | 无需特殊处理，一切流畅 |
| 10K ~ 100K | 涂选时使用 `vtkCellLocator`（O(log n) 查询） |
| 100K ~ 1M | 考虑 LOD 降低非焦点几何体的细节、使用硬件选择器 |
| > 1M | 涂选时仅对可见面操作（视锥裁剪）、分块加载 |

### 2.2 内存管理

- **BoundaryLabel 数组**：每个三角面 1 个 int（4 字节），100 万面 ≈ 4 MB，完全可接受
- **撤销栈**：每个快照 4 MB（100 万面），50 步上限 ≈ 200 MB → 超过时丢弃最旧的
- **多几何体并发**：每个 body 独立的 `vtkPolyData` + `vtkActor`，VTK 数据共享机制自动优化

## 3. 测试设计

### 3.1 测试结构

```
tests/
├── __init__.py
└── paint_tests/                          # 画笔系统 & 场景序列化测试
    ├── __init__.py
    ├── conftest.py                       # 共享 fixtures
    ├── test_roundtrip.py                 # 保存/加载往返测试
    ├── test_save_project.py              # save_project 序列化验证
    ├── test_scene_writer.py              # SceneWriter 统一 B_*.stl 导出
    └── test_triangle_hash_matcher.py     # 三角形哈希匹配器单元测试
```

### 3.2 共享 Fixtures（conftest.py）

| Fixture | 描述 |
|---------|------|
| `parent_poly` | 三角化单位立方体（12 三角面），用作 VTK 测试基础网格 |
| `scene_dir` | 临时目录，含 `S_FOAM.stl`、`B_LAT.stl`（cell 0-3）、`B_TOP.stl`（cell 4-5）、`scene.txt`；测试后自动清理 |
| `scene_dir_with_json` | 在 `scene_dir` 基础上添加 `.stardis_project.json`（含 `zone_parent_map` 和 `body_zone_ids`） |

辅助函数：`_make_triangulated_cube()`、`_extract_cells(poly, cell_ids)`、`_write_stl(poly, path)`

python -m pytest tests/paint_tests/ -v

python -m pytest tests/paint_tests/test_roundtrip.py -v
